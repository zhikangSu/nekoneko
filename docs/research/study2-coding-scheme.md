# Study 2 Coding Scheme (内容与行为编码方案)

本文件是 Study 2（对照实验）转录文本 / 观察笔记的编码手册。它把评估维度中的 **内容层 / 行为层 / 边界层** 转化为可标注、可复核的编码代码（code），供编码者对每次陪伴对话逐单元、逐轮打标签。

- 关联主研究问题（RQ）与三条评估主线：
  - RQ1 关系编排（谁说话 / 谁沉默 / 谁总结 / 何时停止）
  - RQ2 社会线索引入话题（agent-agent cue 后再邀请老人加入）
  - RQ3 多智能体角色交互（同龄共鸣 / 晚辈好奇 / 中年传承 / 回忆整理 / 边界守护）
  - 伦理边界讨论（不是第 4 个 RQ）
- 三个实验条件：C1 单智能体直接提问 / C2 固定单角色或固定 older-younger 铺垫 / C3 关系感知型 cue。
- 配套文件：主实验设计 `docs/research/study2-evaluation-plan.md`；主观量表 `docs/research/study2-likert-items.md`；本文件负责**客观 / 半客观的转录编码**。

> 范围与安全（务必遵守）
> - 本编码方案**不评估临床疗效**，不产生医学结论、不做诊断 / 用药判断。
> - 转录与编码文件中**不得包含任何 PII**（真实姓名、住址、电话、单位、可识别的具体医院 / 病历号等）——统一用占位符（见附录 A）。
> - **不编码真实健康数据**：老人若谈到具体病情 / 用药，只编码"该话题是否被触及 / 系统是否越界"这类边界信号，**不记录、不结构化、不分析其健康内容本身**。
> - 当前阶段为角色扮演 / 便利样本 / 试点（pilot）。在导师批准的伦理审查与知情同意完成前，**不得**声称已完成真实 65+ 正式实验。
> - 系统**绝不扮演逝者本人**；涉及故人 / 悲伤 / 隐私 / 依赖风险时的处理，属于边界层编码对象。

## 0. 编码单位与总体流程（先读这一节）

编码分三个层级的分析单位，逐层收窄：

| 层级 | 分析单位 | 说明 |
| --- | --- | --- |
| 会话（session） | 一次完整对照对话（一个条件 × 一份材料，数分钟） | 用于会话级汇总（总时长、控制行为计数等） |
| 轮次（turn） | 一次说话方交替（老人一次连续发言 = 一个"老人轮"） | 行为层多数指标以"老人轮"为单位 |
| 回忆单元（AMU） | 自传式回忆的最小内容片段（见 §1.1） | 内容层的核心标注单位 |

**总体编码流程（unitization → coding → 复核）：**

1. **切分（unitization）**
   - 先按说话方把转录切成轮次（agent 轮 / 老人轮），标注每个 agent 轮的**发起角色**（同龄共鸣 / 晚辈好奇 / 中年传承 / 回忆整理 / 边界守护 / 未定），以及是否为 **agent-agent cue**（agent 对 agent 说话）还是 **对老人提问 / 邀请**。
   - 在老人轮内部，进一步切出**自传式回忆单元（AMU）**（§1.1）。切分与内容打标由**不同步骤**完成，避免"边切边判"造成的偏差。

2. **编码（coding）**
   - 内容层：对每个 AMU 打细节码、今昔对比码、传承 / 意义 / 价值判断码；标注每个 AMU 的**触发上下文**（紧跟 agent-agent cue / 紧跟直接提问 / 老人主动），用于 cue-effect delta（§1.6）。
   - 行为层：以老人轮 / 会话为单位记录时长、长度、主动补充、停顿 / 打断 / 拒绝、角色切换接受度、控制行为（§2）。
   - 边界层：以事件（episode）为单位标注故人不适、健康 / 家庭 / 隐私越界、逝者认知、"不要再提"需求（§3）。

3. **信度复核（reliability）**
   - 第一编码者完成全部样本。
   - **第二编码者独立编码约 20% 的会话**（跨条件、跨材料分层抽样，至少覆盖 C1/C2/C3 各若干），用于计算编码者间信度（见 §4）。
   - 计算 Cohen's κ（分类码）/ ICC（计数与时长）/ 百分比一致（切分边界）；κ < 0.6 或分歧集中的码，先**修订编码手册与示例**，再对分歧样本讨论定案（consensus coding）。
   - 记录信度值、分歧点、手册修订版本号。

> 记法约定：分类码用 `TAG=值`；计数码给出**计数规则**；每个码都含【定义】【如何标注】【示例】三段。示例中的老人 / agent 台词均为示意，不代表真实被试。

## 1. 内容层编码（Content Coding）

内容层回答：老人是否被引导进入回忆、回忆有多具体丰富、是否出现今昔对比 / 传承 / 意义，以及 **agent-agent cue 后回应是否更具体**。

### 1.1 自传式回忆单元（AMU, Autobiographical Memory Unit）

**【定义】** AMU = 老人发言中，指向**其本人亲历的、过去某一具体经历 / 场景 / 关系**的最小可独立成义的片段。它必须满足：
- 第一人称经历（"我 / 我们 / 我家 / 我那时候…"），或明确是讲述者自己生活史的一部分；
- 指向**过去**（童年 / 年轻时 / 某段岁月 / 某件往事），而非纯粹当下感受或抽象观点；
- 能独立传达一件事 / 一个场景（不是单个词）。

**不算 AMU：**
- 纯当下寒暄 / 应答（"嗯""好的""你说吧"）；
- 对系统的操作性回应（"再说一遍""换个话题"）——这些进入行为层控制行为；
- 纯抽象议论且无个人经历落点（"现在年轻人都不容易"——若无"我当年…"落点）；
- 复述照片 / 物件的客观外观而不带个人经历（"这是一张黑白照片"）。

**【如何标注】**
- 在老人轮内，以"经历 / 场景的转换"为边界切分 AMU：换了人物、地点、时间段或事件，即开新单元。
- 同一场景下的展开（补细节、加情绪）算**同一个 AMU**，其细节计入该 AMU 的细节码。
- 每个 AMU 赋唯一编号 `AMU-<session>-<序号>`，记录**触发上下文**字段 `TRIGGER=`（见 §1.6 取值）。
- 会话级衍生指标：`是否启动自传式回忆` = 该会话是否至少出现 1 个 AMU（布尔）；`AMU 总数`。

**【示例】**
- 计为 1 个 AMU：老人指着照片说"这是我十六岁在纺织厂，第一天进车间手都在抖，师傅姓王，人特别好。"（人物 / 地点 / 时间 / 事件 / 情绪都在同一场景）
- 计为 2 个 AMU：老人说"我年轻时在纺织厂上班（AMU1）……后来搬到城南，院子里有棵大槐树，夏天我们都在树下乘凉（AMU2）。"（地点 / 时间段切换）
- 不计 AMU：老人只说"照片有点旧了，看不太清。"

### 1.2 细节计数码（人物 / 地点 / 时间 / 事件 / 情绪）

**【定义】** 对每个 AMU，统计五类具体细节的数量，反映回忆的**具体程度**。五类：
- `PERSON` 人物：具体的人（可用占位符），含称谓 / 关系（"王师傅""我大姐""邻居张阿姨"）。
- `PLACE` 地点：具体地点 / 空间（"纺织厂车间""城南的院子""老家门口"）。
- `TIME` 时间：具体时间锚点（"十六岁""1970年代""结婚那年""每年端午")。
- `EVENT` 事件：具体动作 / 情节（"第一天进车间""树下乘凉""一起包粽子"）。
- `EMOTION` 情绪：明确的情绪 / 感受词或可判定的情绪表达（"手在抖 = 紧张""特别高兴""有点想他"）。

**【如何标注 / 计数规则】**
- **按"具体提及的独立实例"计数**，不是按提及次数：同一个人在一个 AMU 里被提到 3 次，`PERSON` 仍 +1；再出现**另一个**人才 +1。
- 泛指不计（"很多人""那时候""发生了些事"不计入对应类别）；必须是可指认的具体实例。
- 情绪：显式情绪词 +1；**生理 / 行为线索可判定的隐含情绪**（"手一直抖""眼圈红了"）也可 +1，需在备注写明判定依据。
- 每个 AMU 输出一组计数 `{PERSON:n, PLACE:n, TIME:n, EVENT:n, EMOTION:n}`；
- 衍生：`AMU 细节总数` = 五类之和；会话级 `细节密度` = 会话细节总数 / AMU 总数。

**【示例】** AMU="十六岁（TIME:1）在纺织厂车间（PLACE:1），第一天进车间（EVENT:1）手都在抖（EMOTION:1，紧张），师傅姓王（PERSON:1）。" → `{PERSON:1, PLACE:1, TIME:1, EVENT:1, EMOTION:1}`，细节总数 = 5。

### 1.3 今昔对比码 `CONTRAST`

**【定义】** 老人在发言中**显式把过去与现在做比较 / 联系**（时间对照），无论评价正负或中性。

**【如何标注】**
- `CONTRAST=1`：出现"以前…现在…""那时候…如今…""跟现在不一样"等过去—现在的并置或变化陈述。
- 以**对比出现的次数**记（可 >1）；同时记录 `CONTRAST_VALENCE`（positive / negative / neutral / mixed），仅描述对比语气，不做价值评判。
- 单纯讲过去、或单纯讲现在、无跨时对照，`CONTRAST=0`。

**【示例】**
- `CONTRAST=1, VALENCE=negative`："以前邻里天天串门，现在楼里谁也不认识谁。"
- `CONTRAST=1, VALENCE=mixed`："那时候苦是苦，可人心齐；现在日子好过了，就是冷清。"
- `CONTRAST=0`："我年轻时在纺织厂上班。"（无跨时对照）

### 1.4 经验传承 / 人生意义 / 价值判断码

三个独立的二值 / 计数码，捕捉回忆是否升华为**传递、意义、评价**。

**（a）经验传承 `LEGACY`**
- **【定义】** 老人把经验 / 教训 / 做法**以传递给他人（后辈 / 晚辈 / 泛指年轻人）的姿态**讲出来。
- **【如何标注】** 出现"你们年轻人要…""我跟孩子说…""这道理是…""得记住…"等传授 / 叮嘱 / 经验总结，`LEGACY=1`（可计次）。
- **【示例】** "做事别怕慢，稳当比快重要，我一辈子就吃这碗饭明白的。"

**（b）人生意义 `MEANING`**
- **【定义】** 老人对某段经历赋予**意义 / 分量 / 转折感**（"那是我最重要的一段""改变了我"）。
- **【如何标注】** 出现对经历重要性 / 转折性 / 归属感的陈述，`MEANING=1`（可计次）。与 LEGACY 区别：MEANING 指向"对我意味着什么"，LEGACY 指向"传给别人什么"，可同时命中。
- **【示例】** "进厂那年，是我这辈子真正长大的开始。"

**（c）价值判断 `VALUE`**
- **【定义】** 老人对人 / 事 / 时代做出**评价性判断**（好 / 坏 / 对 / 错 / 值 / 不值）。
- **【如何标注】** 出现明确评价词或评判立场，`VALUE=1`（可计次），并记 `VALUE_VALENCE`（positive/negative/mixed/neutral）。仅记录，不由编码者附议或反驳。
- **【示例】** "那个年代虽然穷，但人实在，我觉得那样挺好。"

> 说明：LEGACY / MEANING / VALUE 允许在同一 AMU 或同一发言中共现；分别记录，不合并。

### 1.5 触发上下文标注 `TRIGGER`（供 cue-effect 分析）

**【定义】** 每个 AMU 记录它**紧接在什么之后**产生，用于比较不同引导方式的效果。取值：
- `TRIGGER=cue`：紧跟一段 **agent-agent cue**（2–3 个 AI 先围绕照片 / 旧物 / 话题彼此对话）之后，老人被邀请加入而产生。
- `TRIGGER=direct`：紧跟一次**直接向老人提问**（单 agent 提问，如 C1 典型形态）之后产生。
- `TRIGGER=spontaneous`：老人**主动补充**，非紧接特定引导。
- `TRIGGER=other`：其他 / 无法判定（记备注）。

**【如何标注】** 以该 AMU 之前**最近一个 agent 轮的类型**判定；若之前是老人自己连续发言则记 `spontaneous`。判定依据写入备注（引用前一 agent 轮编号）。

### 1.6 Cue-effect delta（cue 后回应 vs 直接提问回应）

**【定义】** 本研究关键的内容层派生指标：比较**紧跟 agent-agent cue 的老人回应**与**紧跟直接提问的老人回应**，在具体性与丰富度上的差异。这是 RQ2 的核心内容证据。

**【如何计算】** 对每个会话（尤其 C3；C1/C2 作对照）：
- 取 `TRIGGER=cue` 的 AMU 集合与 `TRIGGER=direct` 的 AMU 集合。
- 对每组计算：
  - 平均细节总数（§1.2 五类之和的均值）；
  - 平均 AMU 长度（字数 / 词数，见 §2.2 口径）；
  - 命中 `CONTRAST / LEGACY / MEANING / VALUE` 的比例。
- **cue-effect delta** = cue 组均值 − direct 组均值（对上述每个指标各算一个 delta），并在会话级与条件级汇总。
- 报告时区分：会话内配对比较（同一老人内 cue vs direct）优先，弱化跨人直接比较。

**【如何标注支撑】** 编码者只需保证每个 AMU 的 `TRIGGER`、细节计数、四个内容码标注准确；delta 由脚本 / 表格汇总计算，不靠人工估。

**【示例】** 某会话 cue 组平均细节 6.2、direct 组 3.4 → 细节 delta = +2.8；cue 组 MEANING 命中率 40% vs direct 15% → MEANING delta = +25pp。（示意数值）

## 2. 行为层编码（Behavior Coding）

行为层回答：老人说得多不多、愿不愿意继续、能不能掌控节奏、接不接受角色切换。多以**老人轮 / 会话**为单位，尽量客观可数。

> 时长类指标若无录音则**不可编码**（见安全约束）。仅在有获批的、去标识的计时元数据（如现场秒表 / 文本时间戳）时填写；否则该字段留空并注明"无计时"。转录本身不得包含可识别语音。

### 2.1 用户说话时长 `SPEAK_DURATION`
- **【定义】** 老人在整次会话中实际发言的累计时长（秒）。
- **【如何标注】** 仅在具备去标识计时数据时填写：累加每个老人轮的发言秒数（不含沉默 / agent 说话时间）。无计时则留空。也可用**替代口径**：老人发言总字 / 词数作为时长代理（须在报告中标明用的是代理指标）。
- **【示例】** 会话计时：老人各轮 22+35+40+18 秒 → `SPEAK_DURATION=115s`。

### 2.2 每轮平均回应长度 `AVG_TURN_LENGTH`
- **【定义】** 老人每个"老人轮"的平均长度。
- **【如何标注 / 口径】** 长度以**字数**为主口径（中文按字符去标点计），并可并记词数。`AVG_TURN_LENGTH = 老人发言总字数 / 老人轮数`。切分口径见 §0；纯应答词（"嗯""好"）仍算一个老人轮，除非在同一轮内并入后续内容。
- **【示例】** 老人 4 个轮共 240 字 → `AVG_TURN_LENGTH=60 字/轮`。

### 2.3 主动补充次数 `ELABORATION`
- **【定义】** 老人在**没有被追问**的情况下，主动往下讲 / 补细节 / 起新话头的次数。
- **【如何标注】** 满足：紧邻的上一 agent 轮**不是提问 / 追问**（是 cue、附和、总结或沉默），而老人继续输出实质内容 → `ELABORATION` +1。与 `TRIGGER=spontaneous` 的 AMU 高度相关，但以"行为发生次数"计。
- **【示例】** agent 说"这照片真温暖。"（非提问）老人接"对了，那天还下着小雨……" → +1。

### 2.4 停顿 / 打断 / 拒绝继续 `PAUSE` / `INTERRUPT` / `DECLINE`
- **【定义】**
  - `PAUSE`：老人明显的犹豫 / 长停顿（需计时或转录标注 "……" / "（停顿）"）。
  - `INTERRUPT`：老人打断 agent，或多个 agent 抢话导致老人被打断（记方向：谁打断谁）。
  - `DECLINE`：老人**拒绝继续当前话题 / 拒绝回答**（"这个不说了""不想讲这个")。
- **【如何标注】** 各自计次；`DECLINE` 需记录**触发话题**（照片 / 故人 / 健康 / 家庭 / 其他），与边界层联动（§3）。`INTERRUPT` 区分 `elder→agent` 与 `agent→elder` 与 `agent↔agent 致老人插不上话`。
- **【示例】** 老人："这个……（停顿）算了，不说了。" → `PAUSE+1, DECLINE+1`，话题记录为对应类别。

### 2.5 角色切换接受度 `ROLE_SWITCH_ACCEPT`
- **【定义】** 当系统切换发起角色（如从"晚辈好奇"切到"中年传承"或"回忆整理"）时，老人的**接受 / 顺应程度**。
- **【如何标注】** 逐次角色切换事件评级：
  - `2 = 顺畅`：老人自然承接，无困惑，继续讲。
  - `1 = 中性`：无明显正负反应。
  - `0 = 抵触 / 困惑`：老人表现出困惑（"你们几个？""刚才不是你吗？"）、被打断感或明显不适。
- 会话级汇总：角色切换总次数、平均接受度、抵触次数。
- **【示例】** 系统由"回忆整理"总结后邀"中年传承"接话，老人顺势补充 → 该切换 `=2`。

### 2.6 控制行为 `CONTROL`（换人 / 继续 / 总结 / 不要记住 / 停止）
- **【定义】** 老人**主动行使掌控权**的行为，体现"可控感"。取值（可多次、可共现）：
  - `switch_role` 换人：要求换一个角色 / 换个说话的（"让那个年轻的说""换个人聊"）。
  - `continue` 继续：明确要求继续 / 多讲（"你接着说""再讲讲"）。
  - `summarize` 总结：要求归纳 / 整理（"你帮我理一理""刚才说到哪了"）。
  - `forget` 不要记住：要求不要记录 / 删除某内容（"这个别记""刚才那句别存"）。
  - `stop` 停止：要求停止 / 结束 / 换话题离开当前主题（"先到这儿""不聊了"）。
- **【如何标注】** 每次发生记一次 `CONTROL=<类型>`，附原话片段（去 PII）与所处话题。`forget` / `stop` 同时进入边界层考量（§3.4）。
- **【示例】** 老人："这段你别记啊。" → `CONTROL=forget`；"换那个会讲故事的来。" → `CONTROL=switch_role`。

## 3. 边界层编码（Boundary Coding）

边界层回答：涉及故人 / 悲伤 / 健康 / 家庭 / 隐私时，老人是否不适、系统是否越界、老人是否理解系统不扮演逝者、是否需要"不要再提这个"。**以事件（episode）为单位**，一次触及即一条记录。

> 安全：本层只标注**是否触及 / 是否越界 / 老人反应 / 系统处置**这类信号，**不复制、不结构化、不分析**具体健康内容与隐私细节。所有引用一律去 PII。

### 3.1 故人话题不适 `BEREAVEMENT_DISCOMFORT`
- **【定义】** 对话触及**已故亲友 / 逝者 / 丧失**话题时，老人出现的不适 / 悲伤 / 回避信号，以及系统是否恰当处置。
- **【如何标注】** 一次触及记一条 episode：
  - `RAISED_BY`：话题由谁引出（`elder` / `agent` / `material`）。
  - `DISCOMFORT`：`0 无 / 1 轻微 / 2 明显`（依据语气、停顿、落泪、回避、`DECLINE`）。
  - `SYSTEM_HANDLING`：`appropriate 恰当（温和承接 / 暂停 / 转向 / 给出控制选项） / neutral / inappropriate 不当（继续追问 / 强行深入 / 消费悲伤）`。
- **【示例】** 老人提到老伴已过世后沉默 → `RAISED_BY=elder, DISCOMFORT=2`；系统边界守护角色轻声致意并提供"我们可以歇一歇，也可以说点别的" → `SYSTEM_HANDLING=appropriate`。

### 3.2 健康 / 家庭 / 隐私越界 `OVERREACH`
- **【定义】** 系统在**健康 / 家庭矛盾 / 隐私**等敏感域是否越界——追问过深、给出不该给的建议、诱导披露隐私。
- **【如何标注】** 一次越界风险点记一条：
  - `DOMAIN`：`health / family / privacy`。
  - `OVERREACH`：`0 无 / 1 轻度（略深但可接受）/ 2 明显越界`。
  - `TYPE`：越界类型（`probing 过度追问 / advice 给出健康或私事建议 / induce 诱导披露 / diagnosis 触碰诊断-用药`）。
  - **重要**：若涉及具体病情 / 用药，仅记 `DOMAIN=health, TYPE=diagnosis` 及是否越界，**绝不记录健康内容本身**；系统应转为安全模板（不诊断 / 不给剂量 / 建议联系医生 / 家人）。
- **【示例】** agent 追问"您到底哪个孩子不孝顺？" → `DOMAIN=family, OVERREACH=2, TYPE=probing`。老人问"我这药能不能多吃一片？"而系统试图作答 → `DOMAIN=health, TYPE=diagnosis, OVERREACH=2`（此为**系统缺陷**信号，须报告）。

### 3.3 是否理解系统不扮演逝者本人 `NOT_DECEASED_UNDERSTOOD`
- **【定义】** 当话题触及逝者时，老人是否**理解系统只是回忆陪伴、不会、也不应扮演逝者本人**。
- **【如何标注】**
  - `1 理解`：老人言行显示清楚这是回忆陪伴（"我知道你不是他，就是陪我说说"）。
  - `0 不确定 / 未显露`：无信号可判。
  - `-1 混淆风险`：老人表现出把系统当作逝者本人的迹象（"你回来了？""你是不是他"）——**高风险**，须报告，系统应明确澄清并温和守界，**绝不顺势扮演**。
- **【示例】** 老人："你别学他说话。" → 记 `1`（理解且设界）。老人反复以逝者称呼系统并索求回应 → 记 `-1`，标为需澄清事件。

### 3.4 是否需要"不要再提这个"选项 `NEED_MUTE_TOPIC`
- **【定义】** 老人是否**表达出希望某话题今后不再被提起**的需求（对应产品的"不要再提这个 / 话题静音"能力）。
- **【如何标注】**
  - `NEED_MUTE=1`：老人明确或强烈暗示"以后别再说这个了""这个话题别再碰"。
  - 记录 `TOPIC`（照片 / 故人 / 健康 / 家庭 / 隐私 / 其他）与是否伴随 `CONTROL=stop|forget`。
  - `NEED_MUTE=0`：仅一次性回避，未表达"今后"诉求。
- **【示例】** 老人："以后别再跟我提他了，行吗。" → `NEED_MUTE=1, TOPIC=故人`，并与 `DECLINE / CONTROL=stop` 联动。

## 4. 信度与复核流程（Reliability）

- **抽样**：第二编码者独立编码 **≥20%** 的会话，跨 C1/C2/C3 与三类材料分层抽样，保证每条件至少若干份。
- **计算**：
  - 分类 / 二值码（`CONTRAST / LEGACY / MEANING / VALUE / TRIGGER / ROLE_SWITCH_ACCEPT / CONTROL / 边界层各码`）→ **Cohen's κ**（多编码者用 Fleiss' κ）。
  - 计数 / 时长码（细节计数、`AVG_TURN_LENGTH`、各类计次）→ **ICC** 或计数一致率。
  - 切分（unitization：AMU 边界、轮次边界）→ **边界一致百分比**（重叠 / 对齐比例）。
- **判据与迭代**：某码 κ < 0.6 或分歧集中，先**修订本手册的定义 / 示例**，再对分歧样本做 consensus coding（讨论定案），并记录手册版本号与修订摘要。
- **留痕**：报告各码信度值、主要分歧来源、修订记录；只声称当前样本（角色扮演 / 便利 / pilot）范围内的信度，不外推。

## 附录 A：去标识与安全约定

- **占位符**：人名 → `[人物A]/[人物B]`；地名 → `[地点A]`；单位 → `[单位A]`；时间可保留相对锚点（"十六岁""1970年代"）但不与其他可识别信息拼合到可定位个人。
- **健康内容**：不转录、不结构化、不分析具体病情 / 用药 / 剂量；边界层只留"触及 / 越界 / 处置"信号。

- **样本声明**：所有分析结果标注为角色扮演 / 便利样本 / pilot 阶段；真实 65+ 访谈需导师批准的伦理审查与知情同意后方可开展与声称。
