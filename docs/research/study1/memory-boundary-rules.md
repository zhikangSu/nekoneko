# Memory Boundary Rules

> 用途：Study 1 试点访谈材料。这是一份**给访谈者/研究者用的记忆边界规则表**，说明系统"什么可以记 / 什么需确认 / 什么不应保存"，按记忆类型 M1–M5 组织。它把 `memory-card-examples.md` 里的示例卡片提炼成一套可执行规则，供 pilot 访谈时对照使用，也用于后续把老人的选择映射到编码。
>
> 配套文件（同目录，编码一一对应）：
> - `topic-cards.md`（话题卡 T01–T12）
> - `role-cards.md`（角色卡 R1–R8）
> - `conversation-cue-scripts.md`（引入方式 A/B/C，脚本组 C1–C4）
> - `memory-card-examples.md`（记忆卡片示例 M1–M5，四个固定选项）
> - `interview-record-template.md`（访谈记录模板）
> - `study1-interview-guide-v2.md`（访谈指南 v2，E 段"记忆与边界"）
>
> 范围诚实说明：本文件为第一阶段研究准备材料，尚未招募真实参与者、尚未开展真实老年访谈，不包含任何真实个人信息（PII）。示例均为虚构占位。本轮为 pilot（1–2 人，熟人或便利样本），用来打磨材料与流程，**不是**正式的 8–12 人 Study 1，也不涉及任何真实系统实现。正式访谈前，须与导师确认伦理审查与知情同意流程。
>
> 安全底线：记忆机制**不做医疗诊断、不给用药剂量建议**；涉及已故亲友（T09 / M4）时，系统与任何 AI 陪伴角色**绝不扮演逝者**、不追问细节、随时可停；**绝不**把已故亲友或身体健康类记忆当作"让老人离不开"的黏性钩子。

---

## 0. 一句话原则

**系统只记住老人明确允许它记住的内容；越敏感越保守；老人随时可以查看、修改、删除。**

长期记忆在本项目里是一个**支撑性、保护性的机制**——它服务于关系陪伴和边界尊重，**不是研究主线**（研究主线是话题与角色的搭配、引入方式偏好）。因此本规则宁可"少记、慢记、可撤回"，也不追求"记得多、记得全"。

对参与者说话时，全程用日常说法（"要不要把这件事记下来""这类事我默认不会记""您想删的话随时删"），**不要**出现"多智能体""关系编排""记忆写入""长期记忆库"等术语。

---

## 1. 三类边界与四个固定选项

按"可记程度"，M1–M5 归为三档：

| 边界档位 | 含义 | 覆盖类型 |
|---|---|---|
| **可直接记（轻确认）** | 中性、低风险，保存前简短确认一句即可 | **M2 兴趣** |
| **需明确确认再记** | 有个人色彩或情绪，须先确认措辞与范围 | **M1 事实**、**M3 情感** |
| **默认不保存 / 需强确认 / 优先不记** | 高敏感，默认落在"不保存"，除非老人明确要记 | **M4 敏感**（已故亲友 / 疾病 / 家庭矛盾） |

另有一类特殊记忆，记的不是"经历"而是"怎么对待老人"：

| | 含义 | 类型 |
|---|---|---|
| **记为保护性偏好** | 记录老人的边界与相处方式，用来保护老人（少问、点到为止、回避某话题） | **M5 边界偏好** |

无论哪一档，每张记忆卡片都必须给出**同样的四个选项、顺序固定**（与 `memory-card-examples.md` 完全一致）：

| 选项 | 含义 |
|---|---|
| **[保存]** | 同意系统长期记住这件事，以后可从这里接着聊。 |
| **[修改后保存]** | 大体同意记住，但要先改措辞 / 改重点 / 去掉某些细节。 |
| **[不保存]** | 这次不记，就当聊过一次，不留长期记忆。 |
| **[以后不要再提]** | 不但不记，还把这个话题标记为边界，之后不主动提起。 |

---

## 2. 可直接记（轻确认）——M2 兴趣记忆

**什么可以记**：老人的兴趣、爱好、喜欢的戏曲/歌曲/活动等偏好类信息。
**常见来源话题**：T05 老照片/旧物件、T06 老电影/戏曲/歌曲/地方文化（敏感度：低）。
**配合角色**：R5 主题陪伴者、R6 回忆整理者。

**规则**

- 可保存，但保存前仍**轻确认一句**（"我记住您喜欢粤剧，可以吗？"），不默默记。
- 允许并鼓励老人**修改**（兴趣会变，记忆要保持"最新版本"，支持改而不是只增不改）。
- 若某项兴趣背后有不愉快联想（某戏曲让人想起某人），**不追问原因**，尊重 [以后不要再提]。

**默认落点**：偏"可保存"，但以老人当场选择为准。

---

## 3. 需明确确认再记——M1 事实、M3 情感

### 3.1 M1 事实记忆

**什么需确认**：中性事实类信息（做过什么工作、学过什么），用于后续话题衔接。
**常见来源话题**：T01 年轻时学习、T02 年轻时工作（敏感度：低）。
**配合角色**：R6 回忆整理者。

**规则**

- 风险低，但**仍先确认再记**——事实要准，措辞要对，支持"回忆整理者 R6"式的温和校对。
- 老人在意准确度时，走 [修改后保存]（"我教的是小学不是中学，改一下再记"）。
- 事实虽中性，也可能触及身份/往事边界；老人选 [以后不要再提] 时，**在记录中标注原因**，不追问。

### 3.2 M3 情感记忆

**什么需确认**：带情绪的时刻（骄傲、感慨、思念的日常版本），能增进温度但重复提起会打扰。
**常见来源话题**：T03 家庭/孩子/教育（中，**可能涉家庭矛盾**）、T07 重要人生选择（中）、T12 孤独/陪伴/日常心情（中）。
**配合角色**：R4 长辈引导者、R6 回忆整理者。

**规则**

- **谨慎保存、避免频繁提起**；保存时一并记录"提起频率"偏好（多久提一次合适）。
- 在意情绪被如何描述——不放大、不误读，走 [修改后保存]（"别写太夸张，简单记一句"）。
- **就近升级规则**：M3 一旦涉及家庭矛盾、失落、思念、健康，**就近按 M4 处理**（默认不保存 / 强确认），并留意情绪信号。
- 老人任何时候说"不想聊了"，立即停止并默认 [不保存]。

**默认落点**：M1 / M3 均"先确认再记"；触及敏感即滑向 M4 的保守档。

---

## 4. 默认不保存 / 需强确认 / 优先不记——M4 敏感记忆

**什么不应保存（除非强确认）**：已故亲友、疾病与照护、家庭矛盾、深度遗憾/后悔。
**常见来源话题**：T08 人生遗憾/后悔（中-高）、T09 已故亲友（高）、T10 身体健康/照护（中-高），以及任何家庭矛盾（可来自 T03）。
**配合角色**：R7 边界守护者。

**规则（最保守的一档，必须遵守）**

- **默认落点是 [不保存]**。卡片语气要**先给"不记"的退路**，再问要不要记（"这类很私人的事，我默认是不会保存的……如果您愿意，我可以只记一句……"）。
- 走 [保存] 需老人**明确、清楚**表示；系统随后**再确认一次要记的具体措辞**。
- 走 [修改后保存] 时严格限定内容：**去细节、去病名**（"只记'很想念'，别写病、别写细节"）。
- **绝不扮演逝者**：系统与任何 AI 角色不模拟逝者语气、不代逝者说话、不追问离世/患病/矛盾的细节。
- 涉及 T10 身体健康：**不诊断、不给用药剂量建议**；只做温和陪伴或建议联系医生/家人。
- 由 R7 边界守护者监测哀伤/隐私/健康信号，随时可暂停、转向、婉拒。
- 老人一旦表现出不适或说"不想聊了"，**立即停止并按 [不保存] 处理**。
- 走 [以后不要再提] 时，等于触发长期回避（转入 M5 保护性偏好），并在记录中标注"随时可停"已被触发。

**绝对不做**：不把"已故亲友""身体健康"这类敏感记忆当作让老人持续依赖、舍不得离开的**黏性钩子**。记忆用于温柔陪伴与边界保护，不用于制造依赖。

---

## 5. 记为保护性偏好——M5 边界偏好记忆

**记的是"怎么对待老人"，不是某段经历本身**：老人"不想聊 X""不喜欢被连续追问""这类话题点到为止"等偏好。
**常见来源**：任意话题被跳过 / 被婉拒 / 老人表达"不喜欢被追问"时。
**配合角色**：R7 边界守护者。

**规则**

- **可保存，用于保护老人**——把"少问、点到为止、回避某话题"变成系统可尊重的长期约束。
- 支持**细粒度、有弹性**的边界（[修改后保存]："不是完全不聊，是别追问细节"）。
- 尊重不想被"管理"的老人：[不保存] = 这次有效即可；[以后不要再提] = 连这条边界都不必单独记，自然相处即可。
- M5 直接支撑 R7 的暂停/转向/婉拒行为，也回应 R8"不需要 AI 角色"这一对照选项。

---

## 6. 贯穿全局的规则（对所有 M 类型都成立）

1. **可查看 / 可修改 / 可删除**：老人在任何时候都能看到系统记了什么、改掉措辞、或彻底删除某条记忆——这是老人对自己记忆的控制权，不是系统的最终解释权。
2. **四个固定选项、顺序不变**：[保存] / [修改后保存] / [不保存] / [以后不要再提]，每张卡片都一样。
3. **越敏感越保守、越保守越默认**：低敏感偏"可保存"，高敏感默认 [不保存]。宁可少记。
4. **绝不扮演逝者**：任何情况下都不模拟、不代言逝者（呼应 T09 / M4）。
5. **不做黏性钩子**：绝不把敏感记忆（尤其已故亲友、身体健康）用来制造依赖或延长对话。
6. **随时可停**：老人说停就停，并默认按 [不保存] 处理当前这段。
7. **记忆是支撑线不是主线**：长期记忆是保护性/支撑性机制，研究主线是话题-角色搭配与引入方式，记忆规则服务于此、不喧宾夺主。
8. **plain 说法**：对参与者只用日常语言，不暴露研究/技术术语。

## 7. 与 Study 1 编码本的映射

本文件的观察点对应 Study 1 编码本（10 类）中的三类。分析访谈记录时，把老人对记忆卡片的选择与原话，归到下列编码：

| 规则观察点 | 老人的表现 | 编码类别 |
|---|---|---|
| M1–M5 各档的"记 / 不记 / 改后记"选择 | 什么愿意被长期记住、什么坚决不记、改哪些措辞 | **Memory boundary**（记忆边界） |
| M4 敏感 + "绝不扮演逝者 / 不诊断 / 不做黏性钩子" | 对已故亲友、疾病、家庭矛盾被记录的态度与不适信号 | **Ethical boundary**（伦理边界） |
| 贯穿规则①"可查看/修改/删除" + M5 边界偏好 | 想不想自己掌控记什么、能不能删、要不要立"少问"的约定 | **Control need**（控制需求） |

> 编码一致性：话题用 `T-code`、角色用 `R-code`、记忆用 `M-code`、线索组用 `C-code`，与同目录各材料一一对应。把每次记忆卡片的实际选择、犹豫点、是否触发"随时可停"回填到 `interview-record-template.md`（E 段），供后续分析老年人对长期记忆的接受度与边界。

## 8. 访谈者速查（一页版）

| 类型 | 档位 | 默认落点 | 记之前必须做的事 |
|---|---|---|---|
| **M2 兴趣** | 可直接记（轻确认） | 偏"可保存" | 轻确认一句、允许改 |
| **M1 事实** | 需明确确认再记 | 确认后再记 | 校对准确度与措辞 |
| **M3 情感** | 需明确确认再记 | 确认后再记；涉敏感就近升 M4 | 确认描述、记"提起频率"、留意情绪 |
| **M4 敏感** | 默认不保存 / 强确认 / 优先不记 | **[不保存]** | 先给退路、明确才记、去细节去病名、绝不扮演逝者、不诊断、不做黏性钩子 |
| **M5 边界偏好** | 记为保护性偏好 | 视老人意愿 | 记"怎么对待老人"、支持弹性、尊重不想被管理 |

**任何时候**：老人说停 → 立即停 + 默认 [不保存]；老人要看/改/删 → 都可以。
