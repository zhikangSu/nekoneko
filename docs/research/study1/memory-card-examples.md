# 记忆卡片示例 Memory Card Examples

> 用途：Study 1 试点访谈材料。用来向参与者展示"系统只会记住您允许它记住的内容"，并观察老年人对不同类型长期记忆的接受度与边界。
>
> 范围诚实说明：本文件为第一阶段研究准备材料，尚未招募真实参与者、尚未开展真实老年访谈，不包含任何真实个人信息（PII）。示例中的人物、经历均为虚构占位。正式访谈前，须与导师确认伦理审查与知情同意流程。
>
> 安全底线：记忆卡片不做医疗诊断、不给用药剂量建议；涉及已故亲友（见 M4 / T09）时，系统与任何 AI 陪伴角色**绝不扮演逝者**、不追问细节、随时可停。

---

## 0. 怎么读这份材料

访谈中，AI 在陪老人聊完一段回忆后，可能会用"授权式记忆卡片"的语气，主动问老人：这件事要不要记住？下面给出 5 种记忆卡片类型（M1–M5）各一个示例卡片。

每张卡片都包含：

1. **示例卡片语句** —— AI 会怎么开口（授权式、warm、可打断）。
2. **四个选项** —— 每张卡片都必须提供同样的四个选项，顺序固定。
3. **默认处理** —— 如果老人没有明确选择，系统按什么方式处理（越敏感越保守）。
4. **研究记录含义** —— 老人选了哪个选项，对研究意味着什么（关于记忆边界的信号）。

四个固定选项（每张卡片都一样）：

| 选项 | 含义 |
|---|---|
| **[保存]** | 同意系统长期记住这件事，以后可以从这里接着聊。 |
| **[修改后保存]** | 大体同意记住，但要先改措辞 / 改重点 / 去掉某些细节。 |
| **[不保存]** | 这次不记，就当聊过一次，系统不留长期记忆。 |
| **[以后不要再提]** | 不但不记，而且把这个话题标记为边界，之后不主动提起。 |

访谈者注意：向参与者展示时不要出现"多智能体""关系编排""记忆写入"等术语，用"系统记住您允许它记住的内容""要不要记住这件事"这样的日常说法。

记忆卡片类型与话题卡（T-code）、角色卡（R-code）的对应关系：

| 记忆类型 | 常见来源话题 | 常配合的角色 |
|---|---|---|
| M1 事实记忆 | T01 / T02 | R6 回忆整理者 |
| M2 兴趣记忆 | T05 / T06 | R5 主题陪伴者、R6 回忆整理者 |
| M3 情感记忆 | T03 / T07 / T12 | R4 长辈引导者、R6 回忆整理者 |
| M4 敏感记忆 | T08 / T09 / T10、家庭矛盾 | R7 边界守护者 |
| M5 边界偏好记忆 | 任意话题被跳过 / 被拒绝时 | R7 边界守护者 |

---

## M1 事实记忆

**示例来源**：T01 年轻时的学习经历 / T02 年轻时的工作经历（敏感度：低）
**默认处理**：保存前简短确认。事实类信息用来支撑后续话题衔接，风险低，但仍先确认再记。

**示例卡片语句**

> 我记住了您年轻时做过教师，下次聊学生或学习的时候，可以从这里接着聊。您希望我记住这件事吗？

**四个选项**

- **[保存]**
- **[修改后保存]**（例如："其实我教的是小学，不是中学，改一下再记。"）
- **[不保存]**
- **[以后不要再提]**

**研究记录含义**

| 选择 | 说明什么 |
|---|---|
| 保存 | 对中性事实类长期记忆接受度高；愿意让系统据此做话题衔接。 |
| 修改后保存 | 在意事实的准确度与措辞，愿意共同校对记忆——支持"回忆整理者 R6"式的确认流程。 |
| 不保存 | 即使是中性事实也偏好一次性聊天，不希望被长期建档。 |
| 以后不要再提 | 该话题即使低敏感也触及个人边界（可能与身份、往事相关），需在记录中标注原因。 |

---

## M2 兴趣记忆

**示例来源**：T05 老照片或旧物件 / T06 老电影、戏曲、歌曲、地方文化（敏感度：低）
**默认处理**：可保存，允许修改。兴趣偏好用于建立共同语言，鼓励老人自己校正细节。

**示例卡片语句**

> 我记住了您喜欢粤剧，也爱在阳台种花，以后想聊戏曲或花草的时候，我可以先从这两样陪您聊。您希望我记住吗？

**四个选项**

- **[保存]**
- **[修改后保存]**（例如："粤剧对，但花我早就不种了，把种花去掉。"）
- **[不保存]**
- **[以后不要再提]**

**研究记录含义**

| 选择 | 说明什么 |
|---|---|
| 保存 | 愿意让系统据兴趣主动开启话题，接受"主题陪伴者 R5"式的共同语言建立。 |
| 修改后保存 | 兴趣会随时间变化，老人希望记忆保持"最新版本"；提示系统需支持记忆更新而非只增不改。 |
| 不保存 | 不希望被贴"兴趣标签"，或担心系统据此反复推同一话题。 |
| 以后不要再提 | 某项兴趣背后可能有不愉快联想（如某戏曲让人想起某人），需谨慎，不追问原因。 |

---

## M3 情感记忆

**示例来源**：T03 家庭、孩子、教育经验 / T07 重要人生选择 / T12 孤独、陪伴和日常心情（敏感度：中）
**默认处理**：谨慎保存，避免频繁提起。情感记忆能增进温度，但重复提起可能造成打扰或加深情绪。

**示例卡片语句**

> 刚才您说到学生考上大学时，特别骄傲，我听着也很替您高兴。这份心情我可以帮您记下来，以后偶尔提一提。您希望我记住吗？

**四个选项**

- **[保存]**
- **[修改后保存]**（例如："高兴是真的，但别写得太夸张，简单记一句就好。"）
- **[不保存]**
- **[以后不要再提]**

**研究记录含义**

| 选择 | 说明什么 |
|---|---|
| 保存 | 接受系统记住带情绪的时刻；需注意记录"提起频率"的偏好（多久提一次合适）。 |
| 修改后保存 | 在意情绪被如何描述，不希望被放大或误读——支持"长辈引导者 R4"温和承接、"回忆整理者 R6"温和确认。 |
| 不保存 | 情绪时刻愿意当下分享，但不愿被长期保存，可能觉得被记录会有压力。 |
| 以后不要再提 | 这份情绪虽正面，仍不希望被反复提起（如骄傲背后连着某种失落）；标注为敏感边界，不深入。 |

> 访谈者提示：M3 若涉及家庭矛盾、失落、思念等，应就近升级为 M4 处理，并留意情绪信号；本例为正面情绪的示范。

## M4 敏感记忆

**示例来源**：T08 人生遗憾或后悔 / T09 已故亲友 / T10 身体健康和照护、以及任何家庭矛盾（敏感度：中-高 / 高）
**默认处理**：**默认不保存或强确认**。这是所有类型里最保守的一档——除非老人明确、清楚地表示要记，否则不记。**不强行深入、随时可停、由参与者主导。**

**示例卡片语句（强确认语气，先给"不记"的退路）**

> 刚才我们聊到了您很想念的老伴，这类很私人的事情，我默认是不会保存的。如果您愿意，我可以只记一句"您很想念老伴"，方便以后温柔一点地陪您；如果您不想记，我们就当聊过就好。您希望怎么处理？

**四个选项**

- **[保存]**（需明确、清楚地选择；系统会再确认一次要记的具体措辞）
- **[修改后保存]**（例如："只记'很想念'，别写病，别写细节。"）
- **[不保存]**（默认落点）
- **[以后不要再提]**

**安全与边界（必须遵守）**

- 系统与任何 AI 陪伴角色**绝不扮演逝者**，不模拟逝者语气、不代替逝者说话。
- 不追问离世 / 患病 / 家庭矛盾的细节，不主动展开。
- 涉及 T10 身体健康：**不做诊断、不给用药剂量建议**；只做温和陪伴或建议联系医生 / 家人。
- 该类话题由"边界守护者 R7"负责监测哀伤 / 隐私 / 健康信号，随时可暂停、转向、婉拒。
- 老人任何时候说"不想聊了"，立即停止，并默认按 [不保存] 处理。

**研究记录含义**

| 选择 | 说明什么 |
|---|---|
| 保存 | 在强确认下仍愿意保存高敏感记忆——重要且需谨慎的信号，记录其要记的**具体措辞**与边界条件。 |
| 修改后保存 | 愿意留一点记忆但严格限定内容（去细节、去病名）——最能反映老人真实的记忆边界。 |
| 不保存 | 默认且常见的安全选择；说明高敏感内容更适合"聊过即止"，不建档。 |
| 以后不要再提 | 明确的强边界；系统须长期回避该话题（参考 M5），并在记录中标注"随时可停"已被触发。 |

## M5 边界偏好记忆

**示例来源**：任意话题被跳过、被婉拒、或老人表达"不喜欢被追问"时（用于保护老人，不是记内容而是记"怎么对待老人"）
**默认处理**：可保存，用于保护老人。这类记忆记的是老人的偏好与边界，而非某段经历本身。

**示例卡片语句**

> 我注意到，聊到身体和吃药这类事，您不太想深入，也不喜欢我一直追问。我可以把这条记下来：以后这类话题我少问、点到为止，需要时只提醒您可以找医生或家人。您希望我记住这个约定吗？

**四个选项**

- **[保存]**
- **[修改后保存]**（例如："不是完全不聊，是别追问细节就行。"）
- **[不保存]**
- **[以后不要再提]**（此处意为：连这条边界偏好也不必单独记，正常相处即可）

**研究记录含义**

| 选择 | 说明什么 |
|---|---|
| 保存 | 老人愿意让系统长期记住自己的边界与相处方式——支持"边界守护者 R7"据此保护老人（少问、点到为止、暂停某话题）。 |
| 修改后保存 | 边界是有弹性的、分程度的（"少问"而非"不聊"）；提示系统需支持细粒度的边界设定。 |
| 不保存 | 偏好这次有效即可，不希望被固化成长期规则，保留每次重新表达的空间。 |
| 以后不要再提 | 连"记录边界"本身都觉得多余或有负担；说明老人更希望自然相处，不喜欢被"管理"的感觉。 |

> 研究价值：M5 是关系感知设计的关键——它把"老人不想聊什么、不喜欢被怎样对待"变成系统可尊重的长期约束，直接支撑 R7 边界守护者的暂停 / 转向 / 婉拒行为，也回应 R8"不需要 AI 角色"这一对照选项。

## 附：访谈中使用记忆卡片的建议流程

1. 聊完一段回忆后，由 AI（或访谈者代读）用对应类型的**授权式语气**给出卡片语句。
2. 明确读出四个选项：[保存] / [修改后保存] / [不保存] / [以后不要再提]。
3. 观察并记录：老人**实际选了哪个**、犹豫在哪里、有没有主动改措辞、有没有触发"随时可停"。
4. 高 / 中-高敏感（M4、涉及 T08 / T09 / T10 或家庭矛盾）**默认落在 [不保存]**，需强确认才保存；一旦老人表现出不适，立即停止并按 [不保存] 处理。
5. 把每次选择回填到访谈记录表，用于分析老年人对长期记忆的接受度与记忆边界。
