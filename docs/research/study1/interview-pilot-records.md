# Pilot Interview Records

> 试访谈记录表（Study 1 预访谈）
>
> 本文件用于记录 **1–2 次预访谈（pilot）**，对象为一位熟悉的长辈或方便召集的参与者。
> 这不是正式的 8–12 人 Study 1，也不涉及任何真实系统实现，仅用于打磨访谈问题、话题与角色材料。
>
> 配套材料（同目录，请按名引用、复用同一套编码）：
> `topic-cards.md`（话题 T01–T12）· `role-cards.md`（角色 R1–R8）·
> `conversation-cue-scripts.md`（引导组 C1–C4）· `memory-card-examples.md`（记忆类型 M1–M5）·
> `interview-record-template.md`（正式访谈记录模板）· `study1-interview-guide-v2.md`（访谈指南）。

---

## 0. 填写与安全须知（务必先读）

**填写时限：每次预访谈结束后 30 分钟内完成本记录。** 趁记忆新鲜补齐，避免细节流失。

隐私与安全底线：

- **仅用参与者编号**（如 `P1` / `P2`），不写真实姓名、地名、单位、门牌、联系方式等任何 PII。
- **不录音、不录像、不留可识别身份的原始素材**；本表只保留研究者转述与要点。
- 参与者可随时**跳过、暂停或终止**，任何时候都不追问。
- 敏感话题（T08 人生遗憾 / T09 已故亲友 / T10 身体健康照护 / 家庭矛盾）**绝不主动推进**；角色**绝不扮演逝者**；**不做医疗诊断、不给药物剂量**。
- 对参与者**不暴露研究术语**（多智能体 / 关系编排 / 角色间铺垫等仅供研究者内部使用）。
- 正式访谈前，需与导师确认伦理与知情同意流程；本预访谈不对外公开、不承诺任何公开身份。

记录用途：预访谈结论用于修订问题措辞、话题清单、角色卡与引导脚本，并回填到
`study1-interview-guide-v2.md` 与相关材料中。

## 1. 预访谈记录字段说明（通用 8 字段）

每次预访谈都按下表 8 个字段填写。字段固定、顺序固定，方便跨 pilot 比对。

| 问题 | 记录 |
| --- | --- |
| 哪些问题听不懂 | （列出参与者听不懂 / 需重复 / 需换措辞的问题，注明原措辞与建议改法） |
| 哪些话题愿意聊 | （对应 T01–T12，写出参与者主动展开、明显放松的话题及简要原因） |
| 哪些话题明显敏感 | （对应 T01–T12，尤其 T08/T09/T10/家庭矛盾；写出回避、沉默、情绪波动的信号，不追问） |
| 哪些角色自然 | （对应 R1–R8，写出参与者觉得舒服、贴合的角色及场景） |
| 哪些角色尴尬 | （对应 R1–R8，含 R8「不需要 AI 角色」；写出别扭、拒绝或不理解的角色） |
| 多角色短对话是否可接受 | （对应引导组 C 的 C 型「多角色短对话」；记录接受度、困惑点、是否更倾向 A 直接提问或 B 单角色铺垫） |
| 访谈是否太长 | （总时长、参与者疲惫 / 分神的时间点、建议裁剪或合并的环节） |
| 需要删除或改写的问题 | （给出待删 / 待改问题清单，附修改方向；这是回填指南的核心产出） |

> 说明：以上 8 字段与本文件后续每个 Pilot 实例中的表格一一对应，直接在对应实例里填写即可。
