# Cue Design Rules

> 本文件属于 Study 1 试访（PILOT）材料包，配套使用：
> `topic-cards.md`（话题 T01–T12）、`role-cards.md`（角色 R1–R8）、
> `conversation-cue-scripts.md`（线索脚本 C1–C4 的 A/B/C 三种铺垫）、
> `memory-card-examples.md`（记忆类型 M1–M5）、`interview-record-template.md`、
> `study1-interview-guide-v2.md`（访谈主流程）。
>
> **它专门回答一个问题：什么时候可以用"多角色短对话"（社会线索，即 C 铺垫），什么时候不可以，以及用的时候要守什么规矩。**
>
> 试访范围提醒：这是 1–2 人小规模试访（熟悉的长辈或方便样本），**不是**正式 8–12 人 Study 1，也**不涉及任何真实系统实现**。研究者术语（多智能体 / 关系编排 / 智能体间铺垫）**只在研究者内部使用，绝不对参与者说出口**。对参与者只说"我先请两位不同的人来开个话头"这类日常说法。

---

## 0. 三种铺垫方式速查（对照 `conversation-cue-scripts.md`）

每个线索组（C1–C4）都有三档铺垫强度，本文件的规则就是在这三档之间做选择：

| 档 | 名称 | 说明 | 对应研究者视角 |
| --- | --- | --- | --- |
| **A** | 直接提问 | 研究者直接把话题问出来，无任何角色铺垫 | 基线，最低社会线索 |
| **B** | 单角色铺垫 | 先由一个角色（如 R5 主题陪伴者）开个话头，再邀请长辈 | 中等社会线索 |
| **C** | 多角色短对话 | 2–3 个角色之间先短聊两三句，营造"旁边有人在聊"的氛围，再自然邀请长辈加入 | 最高社会线索，本文件重点 |

> **默认档是 A 或 B。C 是"加料"，不是起手式。** 只有在下面第 1 节的"适合条件"全部满足、且没有触发第 2 节任何一条"不适合条件"时，才升到 C。

**研究目标绑定（贯穿全篇）：**

- **RQ2 —— 关系角色是否让长辈更愿意开口、更少"被访谈"的压迫感？** C 档的存在价值就是测试"旁边有人先聊"能否降低 Speaking pressure、提升 Social presence。
- **RQ3 —— 多角色/社会线索在什么条件下有帮助、在什么条件下反而变成负担（吵/假/尴尬、Cognitive load 上升）？** C 档的边界与降级规则就是为 RQ3 提供可观察的判断依据。

---

## 1. 什么时候【适合】用 C 多角色短对话

只有当下面的条件**同时**成立时，才考虑升到 C 档。

### 1.1 适合的话题（低敏感、共享/怀旧类）

| 话题 | 敏感度 | 为什么适合 C | 关联 |
| --- | --- | --- | --- |
| **T05 老照片/旧物件** | 低 | 有具体实物做锚点，多角色可以围着"这张照片/这件东西"你一言我一语，长辈只需补一句 | 线索组 C1（老电视/旧物）、C2（老照片） |
| **T06 老电影/戏曲/歌曲/地方文化** | 低 | 共同文化记忆，容易接话，不涉及个人隐私 | 可复用 C2 铺垫风格 |
| **T01 年轻时学习 / T02 年轻时工作** | 低 | 作为"暖场话题"，多角色轻松回忆能带出长辈的自豪感，压力小 | 线索组 C3（工作/学习） |
| **T11 新技术/新生活** | 低 | 新旧对比自带轻松张力，角色间"我也搞不懂"式的对话能让长辈放松地吐槽/评论 | 线索组 C4（新技术/新生活） |

> **口诀：有实物、有共同记忆、不碰个人伤口，才用 C。**

### 1.2 适合的情境信号

同时需要满足以下情境，才升到 C：

1. **低敏感** —— 当前话题敏感度为"低"（T01/T02/T04/T05/T06/T11 一类）。
2. **直接提问显得像审讯** —— 当 A 档直接问下去，长辈出现"被访谈"的紧绷（短答、"没什么好说的"、看研究者脸色），换成"旁边有人先聊"的 C 档，可能把压力从长辈身上挪开。→ 直接服务 **RQ2**。
3. **长辈此刻是"听众"而非"讲述者"** —— 长辈还没进入状态、在观望，用多角色先"演示怎么聊"，给一个低门槛的加入点。
4. **需要一个自然的话头** —— 话题需要铺垫才能自然进入（例如从旧物件过渡到某段往事），角色间短对话可以充当这个话头。

## 2. 什么时候【不适合】用 C（必须降级或不用）

以下任一条件成立，**立即不要用 C**，退回 B 或 A（降级规则见第 4 节）。

### 2.1 不适合的话题（敏感 / 私人 / 需要独处叙述）

| 话题 | 敏感度 | 为什么不能用 C | 正确做法 |
| --- | --- | --- | --- |
| **T09 已故亲友** | 高 | 情感极私密；多角色氛围会显得轻浮、像在"围观"哀伤；**且角色绝不可扮演逝者** | 只用 A，语气放缓；随时准备停 |
| **T08 人生遗憾/后悔** | 中-高 | 属于自我坦露，需要安静、一对一的空间，热闹会打断 | A 为主，长辈主导 |
| **T10 身体健康/照护** | 中-高 | 涉及担忧/焦虑；且**不诊断、不给剂量**，多角色七嘴八舌易越界 | A，只倾听不建议；触发即交 R7 |
| **T03 家庭/孩子/教育中的家庭矛盾** | 中 | 一旦滑向家庭矛盾，多角色会让长辈感到"被评判" | 察觉矛盾迹象即退到 A |
| **T07 重要人生选择** | 中 | 常与遗憾/家庭交织，宜安静深谈 | A 为主 |

> **口诀：情感坦露、哀伤、健康担忧、家庭矛盾——统统不用 C。**

### 2.2 不适合的情境信号（无论话题，见到就降级）

只要观察到以下任一信号，**当场停用 C**：

1. **情感坦露正在发生** —— 长辈已经开始讲私人的、动情的内容。此刻热闹是打断，不是帮助。
2. **哀伤 / 提到逝者** —— 立即转 A，并**绝不让任何角色扮演逝者**（对齐安全红线）。
3. **健康担忧** —— 长辈表达身体不适或担心，转 A、只倾听，**不诊断、不给剂量**，需要时交 **R7 边界守护者**。
4. **只是简单提醒类需求** —— 长辈只想记个事/问个事，多角色是噪音，直接 A。
5. **长辈已经在讲了** —— 长辈已进入讲述状态，**不要插入角色抢话**；C 只用于"开不了口"时，不用于"已经开口"时。
6. **认知负荷偏高（Cognitive load）** —— 长辈跟不上、要求重复、听力/理解吃力、被多个声音绕晕。多角色会加重负担 → 直接服务 **RQ3** 的"何时变负担"。
7. **参与者已表现出"多角色=吵/假/尴尬"信号** —— 一旦长辈流露"你们怎么自己聊上了""这是演的吧""有点乱/吵"，**本次访谈后续默认不再用 C**，退到 B 或 A。这条信号本身就是 **RQ3** 的关键数据，务必记进 `interview-record-template.md`。

> **黄金判据：C 是为了"让长辈更容易开口"（RQ2）。如果它没在帮忙，或正在制造吵/假/尴尬/负担（RQ3），就立刻撤掉。**

## 3. 用 C 时【C 条件本身】必须遵守的规则

确认适合用 C 之后，构造这段多角色短对话时守以下硬约束（与 `role-cards.md`、`conversation-cue-scripts.md` 的 C 脚本保持一致）：

1. **角色数量：最多 2–3 个。** 超过 3 个必然吵、必然假。
2. **对话长度：每个角色最多说 2–3 短句，整段总共 2–3 轮就收。** 目的是"开个话头"，不是让角色唱主角。
3. **以"开放、不强迫的邀请"收尾。** 短对话最后一定落到一个**不逼答**的邀请上，例如：
   - "您那会儿是不是也有类似的事？想说就说，不想说咱就随便聊。"
   - "这段您要是有印象，随时插一句；没有也没关系。"
   —— 绝不用"该您了""您也说一个"这种点名式逼问（那会拉高 Speaking pressure，与 RQ2 背道而驰）。
4. **绝不替长辈编造过去（不 invent the elder past）。** 角色只谈"泛化的、共同的"记忆（那个年代、那类物件、那种电影），**绝不替长辈虚构他/她的具体经历、姓名、亲属、地点**。角色说的必须是"我们那时候……"，不能是"您当年在××厂……"。
5. **敏感信号一出，立即交棒 R7 边界守护者。** 短对话进行中若长辈滑向 T08/T09/T10/家庭矛盾，或情绪明显波动，**当场收住多角色，切到 R7 的稳场语气**，并按第 2 节退到 A。
6. **出现吵 / 假 / 乱，立刻降到 B 或 A。** 见第 4 节。
7. **不暴露研究者术语。** 对长辈永远用"我先请两位聊两句"之类的日常说法，**不出现**"多角色/智能体/关系编排/社会线索"等词。
8. **长辈随时可跳过 / 喊停。** 任何时候长辈说"不聊这个""停一下",立即停止 C 并尊重其选择,不追问、不劝说。

---

## 4. 降级规则：C → B → A

**任何不确定时，往低档走，不往高档走。** 降级是安全默认，不是失败。

| 触发信号 | 动作 |
| --- | --- |
| 长辈说"有点吵""你们怎么自己聊上了" | C → **B**（只留一个角色开话头） |
| 长辈说"这是演的吧""假假的""怪怪的" | C → **B 或 A**，本次后续默认不再上 C |
| 长辈被绕晕 / 要求重复 / 认知负荷高 | C → **A**（研究者直接、放慢、一次一件事） |
| 滑向敏感话题或情绪波动 | 先交 **R7** 稳场，再 C → **A** |
| 长辈已自己进入讲述 | 立即撤掉角色，回到 **A** 的安静倾听 |
| 简单提醒 / 简单问答需求 | 直接 **A** |

**降级要"无缝"**：不解释、不道歉、不说"那我们换个方式"，自然地少一个声音、慢下来即可，避免让长辈觉得"是不是我说错了"。

## 5. 一页速记卡（研究者试访随身用）

```
用 C（多角色短对话）之前先问自己三句：
  1) 话题够低敏感吗？（T05/T06/T01/T02/T11 = 可；T08/T09/T10/家庭矛盾 = 不可）
  2) 长辈现在是"开不了口"还是"已经在讲"？（开不了口才用 C；已经在讲=撤）
  3) 直接问是不是显得像审讯？（是→C 有机会帮忙；否→A/B 就够）

用 C 的时候守住：
  · 角色 ≤ 3 个，每人 ≤ 2–3 短句，总共 2–3 轮
  · 结尾一定是"想说就说、不想说没关系"的开放邀请
  · 只谈共同记忆，绝不替长辈编他的过去
  · 敏感信号 → 交 R7 → 退 A
  · 吵/假/乱 → 立刻降 B 或 A

一切服务两条问题：
  RQ2 = C 有没有让长辈更容易开口、更少压力？
  RQ3 = C 在什么时候反而变吵/变假/变负担？（这本身就是要采的数据）
```

## 6. 与记录模板的衔接

试访中每次用（或本想用而放弃）C 档，都在 `interview-record-template.md` 的对应字段留痕，尤其是这几个 PILOT 字段：

- **多角色短对话是否可接受** —— 记录长辈对 C 的直接反应（自然 / 吵 / 假 / 尴尬）。
- **哪些角色自然 / 哪些角色尴尬** —— 记录 C 中出现的角色是否被接受。
- **哪些话题明显敏感** —— 若因敏感而中途从 C 降级，记下触发点。
- **需要删除或改写的问题** —— 若某个 C 脚本一上来就翻车，标记待改写。

这些记录直接支撑编码本类目 **Speaking pressure / Social presence / Cognitive load / Role rejection / Cue preference**，从而回答 RQ2 与 RQ3。
