# 关系感知型多智能体老年陪伴：文献矩阵 (Literature Matrix)

本研究围绕老年回忆陪伴，提出三条核心机制：**(1) 关系编排 (relationship orchestration)** —— 由话题、人生经历、关系偏好与边界需求驱动，动态决定"谁说话 / 谁沉默 / 谁追问 / 谁总结 / 何时收束"；**(2) 社会线索引入话题 (agent–agent social cueing)** —— 让 2–3 个 AI 关系角色围绕一张照片 / 一件旧物 / 一个人生话题先行短对话，再邀请老人加入，作为比"直接提问"压力更低的开场；**(3) 多智能体角色交互 (multi-agent role interaction)** —— 让 同龄共鸣 / 晚辈好奇 / 中年传承 / 回忆整理 / 边界守护 等关系角色协作，避免 **吵 / 假 / 乱 / 越界**。主研究问题：*老年回忆陪伴中，系统如何基于话题、人生经历、关系偏好和边界需求，动态编排不同 AI 关系角色，并通过多智能体短对话作为社会线索，更自然地引导老人进入回忆与自我表达？*（RQ1/RQ2/RQ3 分别对应上述三机制；长期记忆 / 信任 / 控制为支撑性评估维度与伦理边界，而非第四个 RQ。）本文献矩阵服务于"**别人怎么做 → 还没解决什么 → 我们如何推进**"的论证链，直接支撑 proposal / 最终报告的 Related Work，覆盖 A–E 共 5 个主题组。

**共收录 28 篇文献**（组 A=6、B=5、C=3、D=9、E=5）。每篇的 **Area（所属主题组）** 字段由其所在的"## 组 X"分节标题承载，其余 8 个矩阵字段（Paper、作者/年份、Research Question、Method、System / Material、Findings、Limitation、Use for Our Study、Extension）逐行给出。分组图例：

- **组 A** — 老年回忆 / life review（回忆的心理功能与干预循证）
- **组 B** — 技术支持回忆（照片 / 旧物 / 音乐 / VR / 对话 agent，含关键基线 ReminiBuddy）
- **组 C** — 老年 conversational / relational agents（关系型对话智能体与长期关系）
- **组 D** — 多智能体编排 / turn-taking / human control（轮次分配、编排框架与人类控制）
- **组 E** — 信任 / 控制 / 透明 / 记忆 / 伦理边界（协作记忆、社会传染、信任校准与 HAI 准则）

## 本研究与 ReminiBuddy 的区别

ReminiBuddy / "Chorus of the Past" 是最直接的对照基线：它用**固定的年长/年轻陪伴智能体** + 个人照片 + 3D 怀旧旧物支持回忆，并发现身份×线索共同塑造体验（年轻智能体更多情感投入、年长智能体更强内容共鸣；个人照片触发自传体记忆、通用旧物触发共时代记忆）。我们在此之上做四点推进，直接回应"我们凭什么不同"这一验收问题：

- **固定角色 → 动态关系编排**：我们不预设"年长/年轻"两个固定身份，而是在 同龄共鸣 / 晚辈好奇 / 中年传承 / 回忆整理 / 边界守护 的角色分类上，按话题、人生经历、关系偏好与边界需求**动态编排**谁说话/沉默/追问/收束（机制1，RQ1）。
- **并列回应（不真正互相说话）→ 被实证检验的 agent–agent 社会线索**：ReminiBuddy 的多个智能体只是**并列回应老人**、彼此并不真正对话；我们让 2–3 个角色**先围绕照片/旧物短对话，再邀请老人加入**，并**实证检验**这种社会线索是否比直接提问更自然（机制2，RQ2）。
- **无角色交互护栏 → 显式角色交互规则**：我们用显式规则让多角色协作时避免 **吵 / 假 / 乱 / 越界**（机制3，RQ3），把"多个声音同时在场"从风险变为可控设计。
- **未处理逝者/哀伤边界 → 边界守护作为伦理维度**：ReminiBuddy 未聚焦逝者/哀伤的边界与伦理；我们把**边界守护**设为独立角色与伦理维度，结合组 E 的社会传染/虚假记忆告诫，防止 AI 断言未经核实的个人往事。

## 研究缺口小结

- **记忆线索的社会化尚未落地（组 B → 机制2）**：组 B 把技术稳定地确立为**静态记忆线索**（照片/旧物/音乐/VR），但几乎无人把"由谁、以何种社会情境发起线索"当作可设计变量。我们据此提出用 **agent–agent 短对话作为社会记忆线索**，填补"素材线索"到"社会线索"的空白。
- **编排重于数量（组 D → 机制1/3）**：组 D 提供了 turn-taking 理论、可编程 speaker selection、置信度轮次与人类控制/trace 工具，但目标多为**任务求解或可信模拟**，speaker selection 由任务而非关系/边界驱动。我们主张关键不在"有几个智能体"，而在**由话题/经历/偏好/边界驱动的编排**与防吵/假/乱/越界的角色交互规则。
- **agent–agent 线索在老年回忆中未被验证（组 D/E → 机制2）**：组 D 展示 agent–agent 交互可生成可信社会交流，组 E 表明协作记忆既能交叉提示、也会带来抑制/虚假记忆风险；但**尚无研究在老年回忆场景中实证检验 agent–agent 对话作为进入回忆的社会线索**，这正是本研究的核心可检验缺口，并须以边界守护规避虚假自传体记忆。

## 如何用于 Related Work / Proposal

组 A/B 铺陈 proposal §4 相关工作的**回忆价值与技术线索基线**，并把 ReminiBuddy 设为直接对照，界定三机制的贡献边界；组 C/D 支撑 **RQ1（关系编排）** 与 **RQ3（角色交互）**——从关系型对话与多智能体编排文献中提取"谁说/沉默/追问/收束"的可操作机制与人类控制视角；组 D/E 交叉支撑 **RQ2（社会线索）**，把 agent–agent 交互与协作记忆理论转化为可检验的社会记忆线索。组 E 同时为长期记忆 / 信任 / 控制 / 伦理边界这一**支撑性评估维度**提供设计清单与告诫，贯穿三机制而不构成第四个 RQ。
