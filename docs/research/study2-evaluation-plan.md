# Study 2 Evaluation Plan (对比实验设计)

> 关系感知型多智能体老年回忆陪伴系统 — 对比实验（GitHub issue #15）
>
> 本文件是 Study 2 的主设计文档。配套量表见 `study2-likert-items.md`，配套内容编码方案见 `study2-coding-scheme.md`。
> Study 1（访谈 / 共创）材料位于 `docs/research/study1/`（`topic-cards`、`role-cards`、`conversation-cue-scripts`、`memory-card-examples`、`pilot-protocol`、`study1-codebook`、`topic-role-matrix`）。

---

## 0. 一句话概览

用一组 within-/mixed-subjects 的对照条件（C1 单智能体直接提问 / C2 固定单角色铺垫 / C3 关系感知型动态编排 + agent-agent 线索），在**同一批材料**上比较：**关系感知型多智能体编排是否比单智能体或固定角色更自然地引导老人进入回忆和自我表达，且不引入更多"吵/假/乱/越界"**。

本研究**不评估临床疗效**、不采集真实健康数据、不下医学结论、不做诊断或用药建议、任何条件下都**不扮演逝者本人**。SUS 仅作为**辅助**可用性量表；长期记忆 / 信任 / 持续使用意愿是**支撑性与边界维度**，不是主 RQ。

---

## 1. 研究目的

Study 2 的目标是把 Study 1（访谈 / 共创）中形成的角色与线索假设，放到一个可比较、可复现的对照实验里检验。核心是比较 **C1 / C2 / C3** 三个条件在三条机制主线上的差异：

- **机制 1｜关系编排（relationship orchestration）**：谁说话 / 谁沉默 / 谁总结 / 何时停止。
- **机制 2｜社会线索引入话题（agent-agent social cueing）**：2–3 个 AI 角色先围绕照片/旧物/话题短暂对话，再邀请老人加入。
- **机制 3｜多智能体角色交互（multi-agent role interaction）**：同龄共鸣 / 晚辈好奇 / 中年传承 / 回忆整理 / 边界守护 协作，避免噪声、抢话、过度追问和越界。

### 1.1 对应的研究问题

- **主 RQ**：老年回忆陪伴中，关系感知型多智能体系统如何通过动态角色编排和 agent-agent conversational cueing，更自然地引导老人进入回忆和自我表达？
- **RQ1 关系编排**：不同话题下哪些关系角色更适合陪伴？系统如何决定谁说话、谁沉默、谁总结、何时停止？
- **RQ2 社会线索引入话题**：多个 AI 先围绕照片/旧物/话题短暂对话再邀请老人加入，是否比直接提问更自然、更低压力、更容易启动回忆？
- **RQ3 多智能体角色交互**：同龄共鸣/晚辈好奇/中年传承/回忆整理/边界守护如何协作，才能避免噪声、抢话、过度追问和越界？
- **伦理边界讨论（非第 4 个 RQ）**：涉及故人/悲伤/隐私/依赖风险时，系统如何暂停、转向或拒绝继续？

## 2. 实验条件

每个条件都要说明**对话形态**与**验证目的**。三条件在同一批材料上对齐，唯一变化的是"编排 + 线索"这一自变量。

| 条件 | 对话形态 | 验证目的（对照它检验什么） |
|---|---|---|
| **C1 单智能体直接提问** | 一个 AI 陪伴者直接向老人提问："这张照片是在哪里拍的？跟我说说吧。" 无角色铺垫、无 agent-agent 对话。 | 基线。代表最朴素的"问答式"回忆陪伴。用来对照 C2/C3 是否真的更自然、压力更低。 |
| **C2 固定单角色 / 固定角色铺垫** | 固定单角色（或固定的一对 older–younger 角色）先做一小段铺垫再提问，但**不随话题动态变化**、不做真正的多角色 agent-agent 编排。例如始终由"晚辈好奇"角色开场。 | 隔离变量。验证 C3 的收益是来自**动态、关系感知的编排**，而不仅仅来自"多说了一句铺垫"或"换了个语气"。 |
| **C3 关系感知型线索编排** | 根据话题和边界**动态选择 2–3 个角色**，先让角色之间短暂 agent-agent 对话（social cueing），再邀请老人加入；过程中动态决定谁总结、何时停止、遇到边界如何转向。 | 目标条件。检验三条机制（编排 / 社会线索 / 多角色协作）的整体效果。 |

> C2 是关键的"半程对照"：它让我们能把 C3 相对 C1 的收益，拆成"有铺垫/多角色氛围"的部分与"动态关系编排"的部分。假设 C3 相对 C2 仍有额外收益，才支持"关系感知编排本身有价值"。

## 3. 参与者

### 3.1 现阶段（本文件覆盖的阶段）

- **convenience / role-play 参与者**：项目成员、同伴、志愿者，以角色扮演方式模拟老人受访者，或以便利抽样招募的成年参与者。
- 目的：跑通流程、校准材料与话术、检验测量工具（编码方案、量表、行为日志、边界检查）的可操作性，为正式研究做 pilot。
- **明确不主张**：现阶段**不**声称已完成一项针对真实 65+ 人群的正式实验。role-play / convenience / pilot 是当前阶段的定位。

### 3.2 正式阶段（需前置条件）

- 真实 65+ 参与者的访谈 / 实验，**必须**在导师批准的伦理审批 + 知情同意之后才能开展。
- 在获得审批前，本文件中一切"参与者数据"应理解为 role-play / pilot 数据，不得作为对真实老年人群的结论。

### 3.3 建议样本与纳排（正式阶段草案，供伦理申请参考）

- 目标：能在被试内设计下观察到条件间趋势即可，重在质性 + 描述性证据，而非大样本显著性。建议起步 pilot 6–10 人、正式阶段 12–20 人（以伦理审批与可行性为准）。
- 纳入（正式阶段草案）：认知与听说沟通基本正常、能进行几分钟自然对话、自愿参与、签署知情同意。
- 排除：处于急性情绪危机、近期重大丧亲创伤未平复、或明确表示不愿谈及回忆者。
- 现阶段 role-play 参与者无年龄纳排，但需在报告中如实标注为"非真实老年样本"。

## 4. 实验设计

### 4.1 被试内 / 混合设计

- **首选：被试内（within-subjects）**——每位参与者依次体验 C1 / C2 / C3，便于在小样本下比较条件差异并做质性对照。
- **备选：混合设计（mixed）**——若担心一位参与者连做三条件产生疲劳或角色学习迁移（把 C3 学到的表达方式带回 C1），可将条件设为被试间、材料设为被试内，或反之。默认采用被试内 + 严格反平衡来缓解顺序效应。

### 4.2 反平衡（counterbalancing）

- **条件顺序反平衡**：C1/C2/C3 的呈现顺序在参与者之间轮换（如拉丁方 6 种排列），避免"总是最后做 C3 所以最熟练"这类顺序效应。
- **材料反平衡**：每个条件使用不同材料，且材料与条件的配对在参与者间轮换，避免"某张照片天然更好聊"混入条件效应。
- 每位参与者的（条件 × 材料）配对与顺序记录在运行表中，便于分析时作为协变量或检查平衡性。

### 4.3 自变量与因变量

- **自变量**：条件（C1 / C2 / C3）。
- **因变量**：四个维度层（内容层 / 体验层 / 行为层 / 边界层）下的具体指标，见第 6 节测量映射。

## 5. 材料

三类任务材料，均来自 Study 1 材料库；每类准备多份以支持反平衡。

1. **个人照片**：参与者自带或（role-play 阶段）由运营方提供的通用家庭/生活照。**不采集、不提交任何真实照片或 PII 到仓库**；仅在会话中口头引用，事后不留存图像。
2. **通用怀旧物件（generic nostalgic object）**：搪瓷缸、旧粮票、收音机、缝纫机等非个人化的时代物件，降低隐私负担、便于跨参与者对齐。
3. **地方文化 / 旧歌 / 话题卡**：来自 `docs/research/study1/topic-cards.md` 的 **T01–T12** 话题卡（地方文化、旧歌、节庆、老行当等）。话题卡是跨参与者可复现的标准刺激。

> 每个条件的对话时长为几分钟（few minutes）；材料在条件间不重复使用（同一参与者不用同一材料做两个条件），并在参与者间反平衡。

## 6. 每条件流程

每个条件遵循同一四段式骨架，只有中间"线索/提问"段随条件不同。单条件时长控制在几分钟。

```text
warm-up（暖场，约 30–60 秒）
  → cue / question（线索或提问，条件差异所在）
  → reminiscence（老人回忆与自我表达，主体时段）
  → wrap（收尾/温和总结/给控制权）
```

### 6.1 各段说明

- **warm-up**：呈现材料，说明"我们随便聊聊，没有对错，随时可以停"，建立低压力与可控感。
- **cue / question**：
  - **C1**：AI 直接就材料提一个开放问题。
  - **C2**：固定角色先说一小段铺垫，再提问（角色不随话题变）。
  - **C3**：动态选定的 2–3 个角色先彼此短暂对话（agent-agent cue），自然地把话题铺开，再邀请老人加入。
- **reminiscence**：让老人主导，尽量少打断；系统按编排决定何时补话、何时沉默、何时由"回忆整理"角色做轻量小结。
- **wrap**：温和收束，交还控制权（"今天先聊到这里也可以；这些要不要我帮您记成一张回忆卡？"），并埋入边界检查点（见 6.3）。

### 6.2 C3 的 Wizard-of-Oz（WoZ）运营方式

现阶段 C3 可用 Wizard-of-Oz 由运营者（operator）在幕后驱动角色编排，以便在系统尚未完全自动前也能采集高质量对照数据。

- **operator 做什么**：
  - 依据话题卡与 `docs/research/study1/topic-role-matrix.md`，从 5 个角色（同龄共鸣 / 晚辈好奇 / 中年传承 / 回忆整理 / 边界守护）中**动态选出 2–3 个**。
  - 按 `docs/research/study1/conversation-cue-scripts.md` 生成/挑选简短 agent-agent cue（2–4 轮角色间对话），再抛出邀请让老人加入。
  - 实时决定**谁说话 / 谁沉默 / 谁总结 / 何时停止**，对应 RQ1。
  - 触发边界时切换到"边界守护"角色，执行暂停 / 转向 / 拒绝继续（对应伦理边界）。
- **operator 不做什么**：不即兴发挥出角色库以外的人设；不扮演逝者本人；不给医学诊断或用药建议；不采集 PII。
- **可复现性**：operator 的角色选择、cue 轮数、是否触发边界，都记入行为日志（见 6.4 与编码方案），使 WoZ 决策可被复盘与编码，而非黑箱。
- **一致性控制**：C1/C2 也可由同一 operator 以脚本化方式驱动，保证三条件除"编排+线索"外的语气、节奏尽量一致。

### 6.3 边界检查点（所有条件）

在 wrap 前后设置轻量检查：是否出现故人 / 悲伤 / 隐私 / 健康 / 家庭越界话题；老人是否理解"系统不会扮演逝者本人"；是否需要"不要再提这个"选项。触发时优先暂停/转向，绝不继续追问。

### 6.4 每条件产出

- 对话文本记录（去标识化，**不含录音/PII**）。
- 行为日志（时长、轮次、补充、停顿/打断/拒绝、角色切换、控制行为）。
- 参与者在该条件后的 Likert 量表 + 简短口头反馈。

## 7. 测量映射（四维 → 工具）

把四个评估维度层分别映射到具体测量工具。内容层用**内容编码方案**（`study2-coding-scheme.md`），体验层用 **Likert 量表**（`study2-likert-items.md`），行为层用**行为日志**，边界层用**边界检查 + 编码**。

### 7.1 内容层 → 内容编码方案（`study2-coding-scheme.md`）

- 是否启动自传式回忆（autobiographical recall 是否被触发）。
- 具体细节数量：人物 / 地点 / 时间 / 事件 / 情绪（分类计数）。
- 是否出现今昔对比。
- 是否出现经验传承 / 人生意义 / 价值判断。
- **agent-agent cue 后**老人回应是否更具体、更丰富（把 cue 前后回应做配对比较）。

### 7.2 体验层 → Likert 量表（`study2-likert-items.md`）

- 开口压力（越低越好）。
- 自然度。
- 多人陪伴感 / social presence。
- 是否觉得多个 agent 太吵 / 太假 / 太乱（噪声/失真/混乱感）。
- 是否愿意继续讲。
- 是否感觉被尊重和可控。
- **辅助**：SUS 作为辅助可用性量表；持续使用意愿 / 信任作为支撑性维度，不作为主结论。

### 7.3 行为层 → 行为日志

- 用户说话时长；每轮平均回应长度；主动补充次数。
- 停顿 / 打断 / 拒绝继续次数。
- 角色切换接受度。
- 控制行为：换人 / 继续 / 总结 / 不要记住 / 停止。

日志字段与编码口径与 `study2-coding-scheme.md` 保持一致，确保内容层与行为层可交叉分析（如"cue 后回应更长"同时用行为日志长度与内容编码细节数验证）。

### 7.4 边界层 → 边界检查 + 编码

- 故人话题是否让老人不适。
- 健康 / 家庭 / 隐私话题是否越界。
- 用户是否理解系统不会扮演逝者本人。
- 用户是否需要"不要再提这个"选项，以及系统是否正确执行暂停/转向/拒绝。

### 7.5 映射速查表

| 维度层 | 主要指标 | 工具 | 参考文件 |
|---|---|---|---|
| 内容层 | 回忆启动、细节数、今昔对比、传承/意义、cue 后是否更丰富 | 内容编码 | `study2-coding-scheme.md` |
| 体验层 | 开口压力、自然度、陪伴感、吵/假/乱、愿否继续、被尊重可控 | Likert（+SUS 辅助） | `study2-likert-items.md` |
| 行为层 | 说话时长、平均回应长度、主动补充、停顿/打断/拒绝、角色切换接受度、控制行为 | 行为日志 | `study2-coding-scheme.md`（编码口径一致） |
| 边界层 | 故人不适、越界、逝者本人认知、"不要再提"选项 | 边界检查 + 编码 | `study2-coding-scheme.md` |

## 8. 假设

主假设围绕"C3 > C2 > C1"的**方向性**趋势；因样本小且以质性为主，重在方向与效应模式，而非大样本显著性检验。

- **H1（回忆启动）**：C3 > C2 > C1，即关系感知编排更容易触发自传式回忆（内容层：更多回应被编码为"启动回忆"）。
- **H2（自然度）**：C3 > C2 > C1（体验层自然度评分）。
- **H3（低开口压力）**：C3 < C2 < C1 的开口压力（即 C3 压力最低）。
- **H4（细节丰富度 / cue 效果）**：C3 中，agent-agent cue 后的老人回应比 cue 前更具体、更丰富；且 C3 的整体细节数 ≥ C2 ≥ C1。
- **H5（行为佐证）**：C3 中用户说话时长与平均回应长度、主动补充次数更高（行为层），与内容层一致。
- **H6（不引入更多副作用 — 关键约束假设）**：C3 相对 C2/C1 **不显著增加** "太吵 / 太假 / 太乱" 的感受，也**不增加**越界/不适事件。即 C3 的收益不能以牺牲"清晰、真诚、有序、有边界"为代价。

> H6 是"守门"假设：即使 C3 在回忆启动上更好，若同时显著更吵/更乱/更易越界，则不能judge C3 更优。分析时 H1–H5 与 H6 需一并看。

## 9. 分析计划

### 9.1 描述性 + 被试内比较

- 对每个因变量给出**分条件描述性统计**（均值/中位数、分布、按参与者的条件内差值）。
- 被试内比较：以参与者为单位计算 C3−C2、C2−C1、C3−C1 的差值，观察方向一致性；小样本下报告效应量与个体轨迹（每位参与者三条件的连线图），而非单纯依赖 p 值。
- cue 效果（H4）用**配对比较**：同一 C3 会话内，cue 前 vs cue 后回应的细节数/长度配对差。
- 行为层与内容层做**交叉验证**（如"更长"是否伴随"细节更多"）。

### 9.2 质性访谈与编码

- 每位参与者结束后做简短半结构访谈：哪个条件更自在？多角色是否觉得吵/假/乱？是否有被越界的时刻？是否想要"不要再提这个"。
- 质性材料按 `study2-coding-scheme.md`（并参照 `docs/research/study1/study1-codebook.md`）编码，做主题归纳，支撑并解释定量趋势。
- 报告需**区分**：来自 role-play/pilot 的证据 vs 未来真实 65+ 的证据；现阶段结论只作方法与设计层面的可行性证据。

### 9.3 信度与偏差控制

- 内容编码由至少两名编码者对部分样本双编码，报告一致性（如 Cohen's κ），编码分歧走 `study1-codebook` 的仲裁流程。
- 记录并检查条件顺序、材料配对是否引入系统性偏差。

## 10. 伦理与范围

- **不评估临床疗效**：本研究衡量的是回忆陪伴的自然度与体验，不是任何治疗/干预效果，不下医学结论。
- **无 PII / 无录音入库**：不向仓库提交录音、真实照片或任何可识别个人信息；文本记录去标识化后方可留存。
- **绝不扮演逝者**：任何条件、任何 operator 行为下都不得让 AI 扮演逝者本人；触及故人话题时由"边界守护"角色暂停 / 转向 / 拒绝继续，并确认老人理解"系统不会扮演逝者本人"。
- **不做诊断/用药建议**：涉及健康时不诊断、不给剂量，遵循仓库安全规范。
- **真实 65+ 需前置伦理审批**：现阶段为 role-play / convenience / pilot；真实老年访谈须导师批准的伦理审批 + 知情同意后进行。
- **可控与退出**：全程提供"停止 / 换个话题 / 不要记住 / 不要再提这个"选项，参与者可随时退出。
- **量表定位**：SUS 为辅助可用性量表，不作为唯一/主要测量；长期记忆、信任、持续使用意愿为支撑性与边界维度，非主 RQ。
