# Study 2 Likert Items (体验问卷题项)

> 关联 GitHub Issue #15：关系感知型多智能体老年回忆陪伴系统评估。
> 本文件配套 `docs/research/study2-evaluation-plan.md`（实验设计）与 `docs/research/study2-coding-scheme.md`（对话编码）。

---

## 0. 使用说明（研究者先读）

- **量表类型**：5 点 Likert，统一锚点为
  **1 = 完全不同意 · 2 = 不太同意 · 3 = 一般 · 4 = 比较同意 · 5 = 完全同意**。
- **填写方式**：每段对话（C1 / C2 / C3 各一段）结束后，由研究者**口头逐题朗读**，老人口头作答，研究者代记；不要求老人自己看屏幕填表。措辞已尽量口语化、短句化，适合朗读。
- **反向计分（Reverse-scored）**：标注为 `[R]` 的题目，分析前需做 `6 - 原始分` 反转，使"分数越高 = 体验越好"方向一致。这些题主要针对"太吵 / 太假 / 太乱 / 越界 / 压力"等负面感受。
- **条件对照**：同一套题在 **C1 单智能体直接提问 / C2 固定单角色或固定older-younger铺垫 / C3 关系感知动态角色 + agent-agent 社会线索** 三种条件后各填一次，用于组内比较（顺序需在被试间平衡）。
- **分组与主次**：
  - **A 主线组（RQ1 / RQ2 / RQ3）** 是核心，直接对应三大机制。
  - **B 体验组、C 边界组** 为重要支撑维度。
  - **D 辅助组（SUS 简版可用性 / 信任 / 持续使用意愿）** 明确标注为**辅助 / 支撑指标，不是主 RQ**，仅用于补充解释，不单独作为结论依据。
- **草稿声明**：以下题项为**草稿**，需在 pilot（角色扮演 / 便利样本 / 预试）后根据可懂度、区分度、天花板效应精简与改写；正式 65+ 访谈须在导师批准的伦理与知情同意流程后进行。
- **安全边界**：本问卷**不评估临床效果**，不采集真实健康数据，不做诊断结论；系统在任何条件下都**不扮演逝者本人**。

**每题字段说明**：`编号` · `中文题目` · `映射（RQ / 维度）` · `计分方向`。

## A. 主线组（对应三大核心机制 · 核心指标）

## C. 边界组（故人 / 健康 / 隐私 / 依赖 · 边界维度）

| 编号 | 题目（1=完全不同意 … 5=完全同意） | 映射 | 计分 |
| --- | --- | --- | --- |
| C-1 | 聊到**已经不在的亲人**时，它处理得**让我觉得舒服、没有被冒犯**。 | 边界 · 故人话题不适 | 正向 |

## E. 单条开放式收尾（非 Likert · 可选）

> 每段条件结束后，可加 1–2 个口头开放题，供编码方案（`study2-coding-scheme.md`）质性分析：
> - "刚才这段，哪一处让您**最想接着往下说**？"
> - "有没有哪一下让您觉得**太吵、太假、或者不太舒服**？"
> - （C3 后加问）"它们**先自己聊几句再请您加入**，和直接问您，您更喜欢哪种？为什么？"

## F. 计分与分析速查

- **方向统一**：所有 `[R]` 题先做 `6 - 原始分` 反转，再进入维度均值。
- **维度得分**：按 A1 / A2 / A3 / B / C / D1 / D2 / D3 分别求均值；A1–A3 为主 RQ 证据，B/C 为支撑，D 为辅助解释。
- **条件比较**：A、B（及适用的 C）在 C1 / C2 / C3 间做组内比较；预期方向 C3 ≥ C2 ≥ C1（尤其 A2 低压力、A3 不吵不假、B2 陪伴感）。
- **样本与解读警示**：当前为 pilot / 便利样本 / 角色扮演阶段，**不做统计显著性强结论**，以趋势 + 质性佐证为主。
- **待办（pilot 后精简）**：核查天花板效应（老人普遍打 5 分）、`[R]` 题是否造成困惑、C 组 NA 比例、每段问卷朗读时长（目标 ≤ 3–4 分钟）。
