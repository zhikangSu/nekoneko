# 关系感知型多智能体老年陪伴系统：下一步任务指南

## 0. 你现在处在哪个阶段

你现在已经有了一个比较完整的研究方案，接下来不是马上做完整系统，而是进入**研究准备和 Study 1 设计落地阶段**。

当前最重要的目标是：

> 把“关系编排、社会线索引入话题、多智能体角色交互”变成可以访谈、可以分析、可以推导原型规则的材料。

也就是说，接下来优先做：

1. 和老师确认研究收束方向。
2. 整理文献矩阵，说明别人怎么做、我们怎么接。
3. 完善访谈材料，包括话题卡、角色卡、脚本和记录表。
4. 做 1-2 个 pilot interview。
5. 根据 pilot 修改问卷和方案。
6. 再进入正式 Study 1。

暂时不要急着做完整 React/FastAPI/LangGraph 系统。系统可以后面做 Wizard-of-Oz 或半自动原型。

## 2. 第一优先级：先发老师确认

### 2.1 你要发什么

先发老师这两个文件就够：

1. `关系感知型多智能体老年陪伴系统_可发老师版.md`
2. `relationship-aware-interview-questionnaire.md`

不建议一次把所有草稿都发过去，太多会让老师不好抓重点。

### 2.2 微信可以这样说

老师您好，打扰您啦。

我根据之前的方向进一步细化了一版研究方案，主要把方向收束到三个机制：关系编排、社会线索引入话题、多智能体角色交互。

我现在的计划是先不急着做完整系统，而是先做访谈和共创，用话题卡、角色卡和几种话题引入方式，了解老年人对不同 AI 陪伴角色和多角色对话的接受度。访谈结果再转化成后续原型的关系编排规则。

我把完整方案和访谈问卷一起发给您，您方便的时候帮我看看方向和方法这样细化是否合适。

### 2.3 你希望老师帮你确认什么

你可以在心里准备这几个问题，老师回复后重点听：

1. 研究问题是否够清楚？
2. 重点放在“关系编排 + 社会线索”是否合适？
3. 访谈对象是否可以从 8-12 位老人开始？
4. 访谈是否需要先做伦理审批？
5. 后续是否要做真实系统，还是先 Wizard-of-Oz 就可以？

### 2.4 完成标准

完成以下任意一种就算完成这一步：

- 老师回复说方向可以继续。
- 老师提出修改意见。
- 老师约你汇报。
- 老师暂时没回复，但你已经把材料发出，可以继续做准备。

## 3. 第二优先级：整理文献矩阵

### 3.1 这一步的目的

不是继续无限找论文，而是把论文变成研究逻辑：

> 别人怎么做 → 他们解决了什么 → 还没解决什么 → 我们怎么做。

### 3.2 文献分组

建议分成五组：

| 组别 | 作用 |
|---|---|
| A 老年回忆与 life review | 支撑为什么做回忆、自我表达和人生意义 |
| B 技术支持回忆 | 支撑照片、旧物、音乐、AI/VR 等线索 |
| C 老年 conversational / relational agents | 支撑 AI 陪伴、长期关系和信任 |
| D 多智能体编排 | 支撑角色分工、turn-taking、用户控制 |
| E 信任、控制与伦理边界 | 支撑记忆授权、隐私、依赖和边界守护 |

### 3.3 每篇文献记录什么

建议用这个表：

| 字段 | 写什么 |
|---|---|
| Paper | 论文名 |
| Area | 属于哪一组 |
| Research Question | 论文研究什么 |
| Method | 用了什么方法 |
| System / Material | 系统、材料或实验条件 |
| Findings | 主要发现 |
| Limitation | 没解决什么 |
| Use for My Study | 对我研究有什么用 |
| Extension | 我怎么在它基础上推进 |

### 3.4 先整理哪些文献

优先整理 15 篇，不要一开始整理 40 篇。

必整理：

1. ReminiBuddy / Chorus of the Past
2. Smart Conversational Agents for Reminiscence
3. Understanding and Co-designing Photo-based Reminiscence with Older Adults
4. RemVerse
5. Lazar et al. technology for reminiscence therapy review
6. Butler life review
7. Webster reminiscence functions
8. Bickmore relational agents
9. Razavi older adults dialogue agent
10. Rajaram collaborative memory
11. Sacks turn-taking
12. Perspectra
13. CrafTeam
14. MultiColleagues
15. YES AND

### 3.5 完成标准

完成 `literature-matrix.md`，里面至少有 15 篇文献，每篇都写出：

- 它做了什么
- 它没做什么
- 我们怎么接

如果老师问“你和 ReminiBuddy 的区别是什么”，你能 1 分钟讲清楚。

## 4. 第三优先级：把访谈材料做成可执行版本

你现在已经有访谈问卷，但正式访谈前还需要把材料拆成可用卡片。

### 4.1 话题卡

新建 `topic-cards.md`。

每张话题卡建议包含：

| 编号 | 话题 | 说明 |
|---|---|---|
| T1 | 年轻时的学习经历 | 学校、老师、同学、考试、学习条件 |
| T2 | 年轻时的工作经历 | 工作单位、同事、技能、困难、成就 |
| T3 | 家庭与孩子 | 教育、家庭责任、代际关系 |
| T4 | 老朋友与邻里 | 朋友、邻居、集体生活 |
| T5 | 老照片或旧物 | 照片、家具、老电器、票证 |
| T6 | 戏曲、歌曲、地方文化 | 熟悉的文化内容 |
| T7 | 重要人生选择 | 搬家、工作、婚姻、转折 |
| T8 | 遗憾或后悔 | 谨慎使用，可跳过 |
| T9 | 已故亲友 | 敏感话题，不主动深入 |
| T10 | 新技术和新生活 | 手机、付款、挂号、网购 |

### 4.2 角色卡

新建 `role-cards.md`。

每张角色卡建议包含：

| 角色 | 给老人的说法 | 研究记录含义 |
|---|---|---|
| 同龄共鸣者 | 像同年代的人，能理解那个年代的生活 | 时代共鸣 |
| 晚辈好奇者 | 像年轻晚辈，好奇但不评价 | 追问细节 |
| 中年传承者 | 像成熟中年人，能理解责任和经验 | 价值总结 |
| 长辈引导者 | 更温和，适合聊人生感受 | 情绪承接 |
| 主题陪伴者 | 对戏曲、地方文化等比较懂 | 主题桥接 |
| 回忆整理者 | 帮您把故事整理成一段文字 | 总结记忆 |
| 边界守护者 | 当话题太累或太私密时提醒暂停 | 风险保护 |
| 不需要 AI | 这个话题不想让 AI 聊 | 拒绝机制 |

### 4.3 三种话题引入脚本

至少准备 3 组，每组包含：

1. 直接提问
2. 单角色铺垫
3. 多角色短对话

优先做这三组：

- 老电视或旧物
- 老照片
- 工作或学习经历

可选第四组：

- 新技术和新生活

### 4.4 记忆卡片

准备 3 张样例：

| 类型 | 样例 |
|---|---|
| 事实记忆 | 您年轻时做过教师 |
| 兴趣记忆 | 您喜欢老电影和戏曲 |
| 情感记忆 | 您提到学生考上大学时很骄傲 |

每张卡片都提供选项：

- 保存
- 修改后保存
- 不保存
- 以后不要再提

### 4.5 完成标准

完成以下 4 个材料：

1. `topic-cards.md`
2. `role-cards.md`
3. `conversation-cue-scripts.md`
4. `memory-card-examples.md`

并且你可以拿着这些材料直接找一个长辈试访谈。

## 5. 第四优先级：做 pilot interview

### 5.1 pilot 的目的

pilot 不是正式研究，不是为了得到结果，而是为了检查访谈设计是否可行。

重点检查：

1. 老人能不能听懂你的研究说法。
2. 话题卡是否自然。
3. 角色卡是否尴尬。
4. 多角色短对话是否太假、太吵。
5. 敏感问题是否会让人不舒服。
6. 访谈时长是否过长。

### 5.2 找谁做 pilot

可以先找：

- 自己熟悉的长辈
- 同学或朋友的家人
- 对聊天比较开放的老人

pilot 先做 1-2 位即可。

### 5.3 pilot 前准备清单

访谈前确认：

- 手机录音能正常使用
- 问卷打开方便
- 话题卡准备好
- 角色卡准备好
- 三组脚本准备好
- 记忆卡片准备好
- 纸笔或电脑记录表准备好
- 已说明可以随时跳过或停止

### 5.4 pilot 时怎么做

建议流程：

1. 用 2 分钟说明研究目的。
2. 问基本信息和日常回忆习惯。
3. 做话题卡排序。
4. 做角色卡匹配。
5. 读 2-3 组话题引入脚本。
6. 讨论记忆卡片和边界问题。
7. 结束时问“哪里听起来奇怪或不舒服”。

### 5.5 pilot 后马上记录

每次 pilot 后 30 分钟内记录：

| 问题 | 记录 |
|---|---|
| 哪些问题老人听不懂 |  |
| 哪些话题老人愿意聊 |  |
| 哪些话题明显敏感 |  |
| 哪些角色最自然 |  |
| 哪些角色最尴尬 |  |
| 多角色短对话是否可接受 |  |
| 访谈是否太长 |  |
| 需要删除或改写的问题 |  |

### 5.6 完成标准

完成 1-2 个 pilot，并产出：

1. `interview-pilot-records.md`
2. 修改后的访谈问卷 v2
3. 修改后的话题卡和角色卡
4. 一页 pilot 总结

## 6. 第五优先级：正式 Study 1 访谈

### 6.1 正式开始前确认

正式访谈前需要确认：

- 老师同意你这样访谈
- 是否需要伦理审批
- 知情同意话术清楚
- 录音和数据保存方式清楚
- 不收集不必要的真实身份信息

### 6.2 招募目标

目标：8-12 位 60 岁以上老年人。

尽量覆盖：

- 60-69 / 70-79 / 80+
- 男/女
- 独居/非独居
- 数字技术熟悉/不熟悉
- 不同职业背景

### 6.3 每位参与者需要保存什么

每位参与者一个文件夹或一组文件：

| 文件 | 内容 |
|---|---|
| `P01-notes.md` | 访谈笔记 |
| `P01-transcript.md` | 录音转写 |
| `P01-topic-role-table.md` | 话题-角色匹配 |
| `P01-cue-ratings.md` | 三种引入方式评分 |
| `P01-summary.md` | 访谈后总结 |

### 6.4 访谈中要特别注意

1. 不要诱导老人说“多 agent 好”。
2. 如果老人觉得奇怪，要追问为什么。
3. 如果老人不喜欢某个角色，这也是重要数据。
4. 如果老人说“一个就够了”，要记录原因。
5. 敏感话题不要强行追问。
6. 关注老人用自己的话描述“谁适合聊什么”。

### 6.5 完成标准

正式 Study 1 完成标准：

- 至少 8 位参与者
- 每位都有完整访谈记录
- 每位都有话题-角色匹配记录
- 每位都有三种引入方式偏好记录
- 每位都有记忆和边界偏好记录

## 7. 第六优先级：访谈分析

### 7.1 先做初步编码

新建 `study1-codebook.md`。

编码维度：

| 编码 | 说明 |
|---|---|
| Topic preference | 愿意聊、看情况、不愿聊 |
| Role preference | 适合什么关系角色 |
| Role rejection | 不喜欢什么角色 |
| Cue preference | 直接提问、单角色、多角色短对话偏好 |
| Speaking pressure | 哪些方式降低或增加开口压力 |
| Social presence | 是否产生多人陪伴感 |
| Cognitive load | 是否觉得吵、乱、假 |
| Memory boundary | 哪些内容可以记或不能记 |
| Ethical boundary | 故人、悲伤、隐私、依赖 |
| Control need | 换人、暂停、总结、删除等需求 |

### 7.2 再做矩阵

核心产出一：`topic-role-matrix.md`

| 话题 | 推荐角色 | 辅助角色 | 不推荐角色 | 原因 | 风险 |
|---|---|---|---|---|---|
| 老照片/旧物 |  |  |  |  |  |
| 工作经历 |  |  |  |  |  |
| 家庭教育 |  |  |  |  |  |
| 老朋友/邻里 |  |  |  |  |  |
| 故人怀念 |  |  |  |  |  |
| 新技术 |  |  |  |  |  |

核心产出二：`cue-design-rules.md`

| 情况 | 是否适合多角色短对话 | 规则 |
|---|---|---|
| 旧物/照片启动 | 适合 | 2-3 句预热后邀请加入 |
| 用户已主动讲述 | 不需要 | 不打断，单角色倾听 |
| 敏感话题 | 谨慎 | 先询问是否继续 |
| 老人觉得吵 | 不适合 | 改为单角色 |
| 新技术解释 | 看情况 | 用旧经验做桥接 |

核心产出三：`memory-boundary-rules.md`

| 内容类型 | 处理方式 |
|---|---|
| 兴趣爱好 | 可保存，可修改 |
| 职业经历 | 保存前确认 |
| 自豪经历 | 谨慎保存 |
| 故人内容 | 默认不保存或强确认 |
| 健康/家庭矛盾 | 默认不保存 |
| 不喜欢的话题 | 可保存，用于避免打扰 |

### 7.3 完成标准

分析完成时，你应该能回答：

1. 哪些话题最适合同龄共鸣者？
2. 哪些话题适合晚辈好奇者？
3. 哪些话题不适合多角色预热？
4. 老人接受 agent-agent cue 的条件是什么？
5. 老人最担心 AI 记住什么？
6. 你的 C3 关系感知多智能体到底怎么运行？

## 10. 每周向老师汇报什么

### 第一次汇报

汇报重点：

- 我已经把研究收束为关系编排、社会线索、多角色交互

## 11. 现在最容易犯的错误

1. **一上来做完整系统**
   现在还不知道老人是否接受这些角色，先做系统容易跑偏。

2. **文献继续无限扩展**
   文献不是越多越好。先把 15 篇核心文献读透，能推出你的方法。

3. **访谈问卷太像考试**
   老年访谈要自然，问题要口语化，允许跳过。

4. **让多个 AI 说太多**
   你的创新是社会线索，不是 AI 群聊。多角色短对话应该短、轻、可退出。

5. **把 AI 设计成家人或逝者**
   这会带来伦理风险。更安全的是“关系功能”，不是扮演真实人物。

6. **只记录喜欢，不记录拒绝**
   老人不喜欢某个角色或话题同样重要，可以帮助你定义边界。

## 13. 最简行动版

如果你现在只想知道今天开始做什么，就按这个顺序：

1. 把可发老师版发给老师。
2. 新建 `literature-matrix.md`，先整理 3 篇最核心文献：ReminiBuddy、Photo-based reminiscence、RemVerse。
3. 新建 `topic-cards.md`，把 10 张话题卡写出来。
4. 新建 `role-cards.md`，把 8 张角色卡写出来。
5. 把访谈问卷读一遍，删掉你自己都觉得绕的话。
6. 找 1 个长辈做 pilot。
7. 根据 pilot 改问卷。

先完成这 7 件事，方向就真正从“方案”进入“研究执行”了。
