关系感知型多智能体老年虚拟陪伴系统 Relationship-Aware Multi-Agent Companions for Older Adults

Thesis

我的创新点不是简单增加更多 agent，而是研究老年陪伴中“关系角色”如何被动态编排。

研究对象：老年人回忆、自我表达与陪伴感（长期信任为支持维度）
核心机制：关系编排 · 社会线索引入话题 · 多智能体角色交互
研究路径：访谈共创 → 原型实验 → 内容编码与体验评估
Research Direction / Multi-Agent HCI 01 / 16
0. Research Positioning

从固定多角色陪伴，走向关系感知型动态编排

研究重点不是“多几个 agent”，而是先理解老年人真正希望谁来陪、哪些话题适合谁、什么内容值得被记住。

Core Claim

老年人需要的不是固定身份的陪伴者，而是随话题、人生经历、情绪状态和关系边界变化的关系角色。

Older adults' needs
→ 角色分类体系 → dynamic orchestration
  • 谁来陪：同龄人、晚辈、中年传承者、长辈或主题人物？
  • 怎么陪：直接提问、先让 agents 短暂对话，还是暂时沉默？
  • 何时停：隐私、故人、强烈悲伤和过度依赖如何设置边界？

4. Research Concepts

从想法转化出六个研究概念

智能体间对话

让多个 agent 先短暂对话，形成社会性记忆线索，而不是直接把问题抛给老人。

social memory cue

长期记忆

持续对话不只是记住偏好，而是减少陌生感、建立熟悉感和信任感。

memory for trust

认知熟悉边界

新内容能否被接受，取决于它能否接上老人的既有经验、身份和兴趣。

bounded personalization

动态关系角色

小孩、中年人、同龄人、长辈不是年龄标签，而是活力、传承、认同、包容等关系功能。

role orchestration

陪伴伦理边界

故人相关话题可以作为边界研究，但不应让 AI 直接扮演逝者本人。

memorial guidance

用户控制

老人不需要配置复杂团队，但需要能说“换人、继续、总结、不要记住、停止”。

lightweight control

5. Research Gaps

现有研究还没有解释“关系如何长期被编排”

固定年龄身份

older / younger 是有价值起点，但小孩、中年人、同龄人、长辈代表的是不同关系功能。

agent 间对话未验证

ReminiBuddy 提到潜力，但尚未把 agent-agent conversation 作为社会性记忆线索来测试。

认知熟悉边界缺席

新内容能否被接受，取决于能否接上老人的既有经验、身份和兴趣。

长期记忆不足

多数系统是单 session；如何记住关键内容、延续关系、建立信任仍未充分研究。

伦理边界不足

故人、隐私、悲伤、依赖风险不能简单交给“更拟人”的 agent 处理。

7. Core Insight

同一个老人，在不同话题下需要不同关系

Scene 01

年轻时的学习、工作、奋斗

更适合同龄人或中年传承者，因为它们能提供时代共鸣和价值承接。

Scene 02

家庭、孩子与教育经验

更适合晚辈或中年人，让老人愿意讲述、传授并总结人生经验。

Scene 03

人生遗憾与关系反思

更适合同龄人或长辈型 agent，降低被评价感，进入更深层反思。

Scene 04

新科技、新生活变化

可以由晚辈或解释型 agent 引入，但必须先连接老人已有经验。

Scene 05

故人、丧失与怀念

应使用纪念引导 agent，而不是扮演逝者本人；用户可随时停止或删除。

9. 核心机制 · 社会线索引入话题

Inter-agent conversational cueing：先有聊天场，再邀请老人加入

同龄共鸣

看到这个老电视，我想起以前一家有电视，全院子的人都来看。

晚辈好奇

真的会全院子一起看吗？那时候大家是不是还要提前搬凳子？

中年传承

这不只是看电视，也像是邻里关系的一部分。叔叔/阿姨，您那时候有没有类似经历？

Hypothesis

相比 agent 直接提问，短暂、受控的 agent-agent 对话能够降低开口压力，提供更多回忆入口，并增强多人陪伴感。

Direct question → answering
shared scene → joining
  • 普通 single-agent：直接问“您以前看过这种电视吗？”
  • 固定多角色：older / younger 轮流回应，但角色不随话题改变。
  • 本研究：agents 先搭建社会性线索，再自然邀请老人讲述。

10. 支持维度 · 授权式长期记忆与边界

长期记忆是支持机制与伦理边界，不是并列的核心机制

记忆类型 示例 使用方式 主要风险
事实记忆 “您年轻时做过教师。” 下次聊教育、学生、课堂时自然接上。 隐私泄露、记错事实。
情感记忆 “您谈到学生考上大学时很骄傲。” 后续回忆中温和引用，支持自我表达。 情感操纵、过度拟人。
关系偏好记忆 “您喜欢同龄人陪聊，不喜欢连续追问。” 调整角色、节奏和提问密度。 过度个性化、兴趣固化。

认知熟悉边界

不要直接问“你怎么看 AI 大模型”，而是问“现在学生用 AI 学习，和您以前教学生有什么不同？”

授权式记忆卡片

“我记住了您年轻时做过教师。下次聊学生或学习时，可以从这里接着聊。您希望我记住这件事吗？”

11. System Proposal

系统框架：前台陪伴角色 + 后台调度 agent

前台可见角色

同龄共鸣、晚辈好奇、中年传承、长辈/导师型、主题人物。

出现原则

这些角色不一定同时出现；每轮最多 2–3 个，避免增加认知负担。

用户控制

继续、换人、总结、不要记住、删除记忆、暂停话题。

01 Orchestration

关系编排 Agent

判断当前话题适合哪个角色介入、谁沉默、是否需要切换。

02 Memory

长期记忆 Agent

记住重要经历、关系偏好、敏感话题和喜欢的陪伴方式。

03 Boundary

兴趣边界 Agent

判断新内容是否在认知熟悉范围内，是否要先连接旧经验。

04 Ethics

边界与伦理 Agent

判断亡者、强烈悲伤、隐私、过度依赖等不适合继续的话题。

05 Reflection

回忆整理 Agent

整理故事，生成可保存、可删除、可分享的回忆卡片。

输出

多角色对话、社会性记忆线索、回忆摘要、记忆卡片、透明提示。

后台 trace

记录 role trace、memory trace、topic trace、safety trace，用于诊断和迭代。

边界策略

使用纪念引导 agent，而不是亡者复刻 agent；先保护，再陪伴。

13. Evaluation

把“更好陪伴”拆成可测指标

回忆深度

  • 自传式记忆数量
  • 具体细节数量
  • 人物 / 地点 / 时间 / 情绪细节
  • 公共记忆与今昔对比
  • 主动发起新话题次数

自我表达

  • self-disclosure depth
  • 是否表达感受
  • 是否表达价值判断
  • 是否讲出遗憾、骄傲或人生反思
  • 对 cueing 的回应长度

陪伴与信任

  • perceived companionship
  • social presence
  • 是否觉得 agent 理解自己
  • 是否觉得系统记得自己
  • 持续使用意愿

控制与边界

  • 是否能选择 / 切换角色
  • 是否知道系统记住了什么
  • 多个 agent 是否混乱或打扰
  • 隐私担忧与被监视感
  • 故人话题不适感

14. Contributions

预期贡献：从预设角色走向基于需求的关系编排

C1

老年人陪伴角色需求分类体系

不是直接预设 older / younger，而是先理解老年人希望谁来陪、哪些话题适合谁。

C2

动态关系角色编排机制

根据用户背景、话题、长期记忆和关系需求决定谁来陪、何时切换、何时沉默。

C3

agent-agent conversation as social memory cue

把 agent 间对话从系统装饰转化为面向老年人的社会性回忆入口。

C4

长期记忆与伦理边界设计

研究哪些记忆建立信任，哪些记忆造成侵犯；故人话题使用纪念引导而非复刻。

下一步计划

Step 01

访谈与共创：建立角色需求分类体系、话题—角色匹配矩阵和记忆授权规则。

Step 02

Wizard-of-Oz 原型：先控制 agent-agent cueing 的话术和参与数量。

Step 03

小规模 pilot：比较 C1 / C2 / C3 对回忆启动、陪伴感和认知负担的影响。

Closing

先建立角色需求分类体系，再验证动态编排和社会性记忆线索是否真的有效。
