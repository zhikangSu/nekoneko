# 关系感知型多智能体老年陪伴系统研究方案

## 1. 研究题目

**关系感知型多智能体老年陪伴系统：面向回忆、自我表达与长期信任的动态角色编排**

英文暂定：

**Relationship-Aware Multi-Agent Companions for Older Adults' Reminiscence, Self-Disclosure, and Long-Term Trust**

## 2. 研究概述

本研究关注老年虚拟陪伴场景中，AI 系统如何根据老年人的话题内容、人生经历、关系偏好、长期记忆和敏感边界，动态编排不同的 AI 陪伴角色，从而更自然地支持老年人的回忆、自我表达、陪伴感与长期信任。

不同于普通单一 AI 陪伴者或固定角色的多智能体系统，本研究重点不在于增加 agent 数量，而在于研究**关系角色如何被动态编排**：谁来陪、谁先说、谁追问、谁总结、谁保持沉默、什么时候邀请老人加入、什么时候停止或转向。

本研究进一步关注一种新的话题引入方式：**社会线索引入话题**。也就是说，系统不直接问老人“您想起什么了吗”，而是让两个或三个 AI 角色先围绕照片、旧物或生活话题进行短暂对话，形成一种轻量的多人聊天场，再邀请老人加入。该机制希望降低老人的开口压力，提供更多回忆入口，并增强多人陪伴感。

## 3. 研究背景

回忆并不只是老年人的怀旧行为。已有老年学和心理学研究表明，回忆与 life review 可以帮助老年人进行身份整合、情绪调节、经验传承、社会连接和人生意义建构。技术支持回忆的相关研究也表明，照片、旧物、音乐、地点、虚拟场景和对话都可以作为记忆线索，帮助老年人进入回忆过程。

然而，传统回忆活动往往依赖真实的人类陪伴者，例如家人、朋友、同龄人或照护者。这些陪伴资源并不总是稳定可得。与此同时，现有 AI 陪伴系统常以单一身份直接提问，容易出现以下问题：

1. **关系不匹配**：不同话题适合不同的倾听者。例如工作经历可能更适合同龄人共鸣，家庭教育经验可能更适合晚辈好奇提问，人生选择和遗憾可能需要更温和的引导角色。
2. **话题进入方式生硬**：AI 直接问“您想起什么了吗”可能让老人感到压力，也可能缺少自然的社会语境。
3. **多角色缺乏编排机制**：多个 AI 角色如果同时出现、轮流发言或过度追问，可能造成认知负担，甚至削弱陪伴感。
4. **长期记忆与敏感边界需要谨慎设计**：系统记住用户经历可能增强熟悉感和信任感，但也可能带来隐私、误记、悲伤唤起或情感依赖风险。

因此，本研究希望从“关系编排”的角度重新思考老年 AI 陪伴：老年人需要的不是固定身份的陪伴者，而是能够随话题、情绪、人生经验和边界需求变化的关系角色。

## 4. 相关研究与研究缺口

### 4.1 老年回忆与技术支持回忆

已有 reminiscence / life review 研究说明，回忆活动对老年人的心理健康、自我认同、社会连接和生活意义具有重要价值。技术支持回忆研究进一步探索了照片、旧物、音乐、VR 场景和对话系统在回忆活动中的作用。

例如，photo-based reminiscence 相关研究通过访谈和共创理解老年人如何使用照片进行回忆；RemVerse 通过 AI 辅助虚拟现实支持老年人触发和深化个人记忆；Smart Conversational Agents for Reminiscence 探索了对话 agent 对老年回忆活动的支持可能。

这些研究说明了“回忆线索”的重要性，但它们更多关注**什么材料能够触发回忆**，较少研究**由什么关系角色引入这些线索更自然**。

### 4.2 ReminiBuddy 与多角色回忆陪伴

ReminiBuddy / Chorus of the Past 是本研究最直接的相关工作。该系统使用 older / younger agents，并结合个人照片和 3D 怀旧物件支持老年人回忆。研究发现，不同 agent 身份和不同记忆线索会影响老年人的回忆体验：年轻 agent 可能带来更强的情感吸引，年长 agent 更容易产生内容共鸣；个人照片更容易触发自传式记忆，通用怀旧物件更容易触发时代性共享记忆。

但 ReminiBuddy 仍有可以进一步推进的空间：

1. 角色主要是固定的 older / younger。
2. agents 之间没有真正形成对话。
3. 系统尚未根据老年人的话题、经历和关系偏好动态编排角色。
4. agent-agent conversation 作为社会性记忆线索的效果尚未被充分验证。

本研究希望在此基础上进一步探索：如果 AI 角色不是固定的，而是根据话题、记忆、情绪和边界动态编排，是否能更好地支持老年人的回忆、自我表达和长期信任？

### 4.3 多智能体编排与人类控制

近年来，多智能体系统被用于创意、研究、写作、软件开发和问题解决等场景。Perspectra、CrafTeam、MultiColleagues 和 YES AND 等工作说明，多 agent 系统的关键不只是多个 agent 同时生成内容，而是如何选择、组织和控制不同 agent 的参与方式。

这些研究对本课题有三点启发：

1. 多智能体系统需要**角色差异**，但角色差异必须服务于用户任务。
2. 多智能体系统需要**发言编排**，包括谁发言、谁沉默、谁总结、何时停止。
3. 用户需要保留一定控制权，但老年场景中的控制方式必须低负担、自然、可理解。

因此，本研究将多 agent 编排从创意协作任务转化到老年陪伴任务中，重点研究“关系角色”如何参与老年回忆对话。

### 4.4 研究缺口总结

综合以上研究，可以看到当前仍存在以下缺口：

| 已有研究 | 已解决问题 | 尚未解决问题 | 本研究推进 |
|---|---|---|---|
| 老年回忆与 life review | 说明回忆对老年心理和社会连接有价值 | 没有回答 AI 应以什么关系角色陪伴 | 研究不同话题下的关系角色需求 |
| 技术支持回忆 | 使用照片、旧物、音乐、VR 等线索触发回忆 | 多数线索是物件或视觉材料，社会关系线索较少 | 将 agent-agent conversation 作为社会性记忆线索 |
| ReminiBuddy | 证明 older / younger agents 和记忆物件影响回忆体验 | 角色固定，agents 不真正互相对话 | 设计动态关系编排和多角色互动 |
| 多 agent 编排研究 | 探索专家角色、用户控制和 turn-taking | 多用于创意/任务场景，不面向老年回忆 | 转化为低负担陪伴节奏 |
| 信任与自动化研究 | 说明信任、透明和控制的重要性 | 陪伴 AI 中的长期记忆和情感边界仍需设计 | 加入记忆确认、删除和边界守护机制 |

## 5. 核心研究概念

### 5.1 关系角色

本研究中的关系角色不是让 AI 扮演真实家人或具体熟人，而是指不同的陪伴功能和社会位置。初步设定如下：

| 角色 | 功能 | 典型适用话题 |
|---|---|---|
| 同龄共鸣者 | 提供时代共鸣和共同经历感 | 老照片、旧物、工作、集体生活 |
| 晚辈好奇者 | 以好奇方式追问细节，降低开口压力 | 家庭故事、旧习俗、人生经验 |
| 中年传承者 | 承接经验，帮助总结价值和意义 | 工作经历、家庭教育、人生选择 |
| 长辈引导者 | 温和支持反思和情绪表达 | 遗憾、重要决定、人生转折 |
| 主题陪伴者 | 提供共同语言和话题桥接 | 戏曲、地方文化、职业、老电影 |
| 回忆整理者 | 帮助整理故事并生成记忆卡片 | 对话结束、故事总结 |
| 边界守护者 | 识别隐私、悲伤和依赖风险 | 故人、家庭矛盾、健康、强烈情绪 |

### 5.2 关系编排

关系编排指系统根据当前话题、用户偏好、长期记忆、情绪状态和边界风险，动态决定：

- 当前由哪个角色说话
- 哪个角色保持沉默
- 是否需要多个 AI 先短暂互动
- 是否追问、共鸣、解释、总结或暂停
- 是否生成记忆卡片
- 是否询问用户保存、修改或删除记忆

### 5.3 社会线索引入话题

社会线索引入话题指让多个 AI 角色围绕某个照片、旧物或话题先进行 2-3 句短对话，再邀请老人加入。

示例：

> 同龄共鸣者：看到这个老电视，我会想到以前谁家有电视，邻居们可能都会过去看。
> 晚辈好奇者：真的会大家一起看吗？那是不是还要提前搬凳子？
> 中年传承者：这不只是看电视，也像是一种邻里关系。叔叔/阿姨，您那时候有没有类似的经历？

该机制希望验证：agent-agent conversation 是否可以成为一种新的社会性记忆线索。

## 6. 研究问题

### 主研究问题

相比单一陪伴 agent 或固定多角色 agent，关系感知型多智能体编排是否能更好地支持老年人的回忆启动、自我表达、陪伴感、信任感和控制感？

### RQ1：关系编排

老年人在不同回忆话题下希望由哪些关系角色陪伴？系统能否根据话题、个人经历和关系偏好动态选择合适角色？

### RQ2：社会线索引入话题

多个 AI 先进行短暂对话，再邀请老人加入，是否比 AI 直接提问更能降低开口压力、提供更多记忆入口，并增强陪伴感？

### RQ3：多智能体角色交互

不同关系角色如何分工与协作，才能避免信息过载，同时支持共鸣、追问、总结和边界保护？

### RQ4：长期记忆与边界

当话题涉及故人、悲伤、隐私或过度依赖风险时，系统应如何解释、暂停、转向或拒绝继续？

## 7. 研究方法设计

本研究计划分为两个阶段：第一阶段进行访谈与共创，建立设计依据；第二阶段构建 Wizard-of-Oz 或半自动原型，进行对比实验。

## 8. Study 1：访谈与共创

### 8.1 研究目的

Study 1 的目标不是直接验证系统效果，而是建立关系编排的设计依据。具体包括：

1. 理解老年人平时如何回忆过去，以及和谁聊更自然。
2. 找出不同话题适合哪些 AI 陪伴角色。
3. 理解老年人是否接受多个 AI 先短暂对话再邀请自己加入。
4. 明确哪些内容可以被系统记住，哪些需要确认或不能保存。
5. 识别故人、悲伤、隐私、依赖等敏感边界。

### 8.2 参与者

计划访谈 8-12 位 60 岁以上老年人。参与者尽量覆盖不同年龄段、性别、居住情况、职业背景和数字技术熟悉程度。

前期优先选择无严重认知障碍、能够独立表达观点的参与者。正式研究前将根据学校或课题组伦理要求准备知情同意、录音说明、数据匿名化和退出机制。

### 8.3 访谈材料

访谈将采用半结构访谈与轻量共创结合的方式。材料包括：

1. **话题卡**：年轻时学习/工作、家庭与孩子、老朋友与邻里、老照片或旧物、地方文化、人生选择、故人怀念、新技术、孤独与陪伴等。
2. **角色卡**：同龄共鸣者、晚辈好奇者、中年传承者、长辈引导者、主题陪伴者、回忆整理者、边界守护者等。
3. **三种话题引入脚本**：直接提问、单角色铺垫、多角色短对话。
4. **记忆卡片样例**：保存、修改后保存、不保存、以后不要再提。

### 8.4 访谈流程

| 阶段 | 内容 | 目的 |
|---|---|---|
| A 基本背景 | 年龄、居住情况、技术使用经验 | 理解参与者背景 |
| B 日常回忆习惯 | 平时什么时候想起过去、和谁讲 | 理解真实回忆场景 |
| C 话题卡排序 | 愿意聊、看情况聊、不太想聊 | 找出话题偏好与敏感边界 |
| D 角色卡匹配 | 为不同话题选择合适角色 | 建立话题-关系角色矩阵 |
| E 引入方式对比 | 比较直接提问、单角色铺垫、多角色短对话 | 验证社会线索接受度 |
| F 记忆与控制 | 判断什么可以记、什么需确认 | 建立记忆授权规则 |
| G 敏感话题边界 | 故人、悲伤、隐私、依赖 | 建立边界守护原则 |

### 8.5 访谈示例问题

1. 您平时会在什么情况下想起过去的事情？
2. 您一般会和谁聊以前的经历？同龄人、晚辈、家人、朋友，还是自己想？
3. 哪些话题您比较愿意讲？哪些话题不太希望别人问？
4. 如果 AI 有不同陪伴角色，您希望它像同龄人、晚辈、中年人、主题专家，还是普通聊天对象？
5. 如果两个 AI 先围绕照片或旧物聊两句，再邀请您加入，您会觉得自然吗？
6. 您希望 AI 记住哪些内容？哪些内容必须先问过您？哪些内容绝对不要记？
7. 如果聊到已经离开的亲人或朋友，您希望 AI 怎么回应？

### 8.6 访谈产出

Study 1 结束后将产出：

1. 老年人陪伴关系角色分类。
2. 话题-关系角色匹配矩阵。
3. agent-agent conversational cueing 的设计规则。
4. 长期记忆保存与删除规则。
5. 敏感话题边界和伦理设计原则。

## 9. Study 2：原型与对比实验

### 9.1 原型形式

Study 2 可以基于 Wizard-of-Oz 或半自动原型进行。前期不需要完整实现所有多 agent 自动编排，可以先通过研究者后台选择角色、控制发言顺序和记录 trace，以降低实现风险。

### 9.2 对比条件

| 条件 | 系统形式 | 验证目的 |
|---|---|---|
| C1 Single-agent | 一个 AI 陪伴者直接提问 | 基础对照 |
| C2 Fixed-role multi-agent | 固定同龄 + 晚辈角色 | 对照 ReminiBuddy 式固定多角色 |
| C3 Relationship-aware multi-agent | 根据话题和用户偏好动态选择角色，并允许 agent-agent 短对话 | 验证关系编排和社会线索机制 |

### 9.3 实验任务

每位参与者围绕 2-3 类材料进行对话：

- 个人照片
- 通用怀旧物件图片
- 老歌、地方文化或旧生活图片
- 工作、家庭、新技术等话题卡

每个条件下进行 5-8 分钟对话。每轮结束后进行简短访谈或评分，了解参与者的自然度、开口压力、陪伴感、控制感和隐私舒适度。

### 9.4 评价指标

内容层：

- 自传式记忆数量
- 具体细节数量：人物、地点、时间、事件、情绪
- 今昔对比
- 自我表达深度：价值判断、遗憾、骄傲、人生总结
- 主动补充次数

体验层：

- 陪伴感
- 社会临场感
- 信任感
- 控制感
- 开口压力
- 认知负担
- 隐私舒适度

行为层：

- 用户说话时长
- 每轮平均回应长度
- 打断或停顿次数
- 角色切换接受度
- 记忆保存、修改、删除行为
- agent-agent cue 后回应是否更具体、更丰富

## 10. 系统机制初步设计

### 10.1 输入信息

系统编排时可考虑以下输入：

| 输入 | 例子 | 作用 |
|---|---|---|
| 当前话题 | 老照片、工作、家庭、故人、新技术 | 判断适合的关系角色 |
| 记忆材料 | 照片、旧物、歌曲、地方文化图片 | 提供回忆入口 |
| 用户偏好 | 喜欢同龄人、不喜欢连续追问 | 调整角色和发言方式 |
| 长期记忆 | 过往职业、兴趣、重要地点 | 提高回应贴合度 |
| 情绪状态 | 开心、沉默、难过、抗拒 | 决定是否追问或放慢 |
| 边界风险 | 故人、隐私、健康、家庭冲突 | 触发边界守护 |

### 10.2 编排输出

系统每轮决定：

1. 选择主发言角色。
2. 是否加入辅助角色。
3. 是否使用 agent-agent cue。
4. 当前回应是共鸣、追问、解释、总结还是暂停。
5. 是否生成记忆卡片。
6. 是否询问用户保存、修改或删除。

### 10.3 发言规则

为了避免多个 AI 抢话，初步规则如下：

| 情况 | 编排规则 |
|---|---|
| 话题刚启动 | 最多 2-3 个 AI 短暂互动，每个 1 句 |
| 老人开始讲述 | 其他 AI 保持沉默，由一个角色接话 |
| 老人讲得很具体 | 晚辈好奇者追问 1 个细节，不连续追问 |
| 老人讲到经验或价值 | 中年传承者总结，但要询问是否准确 |
| 老人沉默或回答很短 | 换低压力问题，或提供可选入口 |
| 老人表达不适 | 立即停止追问，询问是否换话题 |
| 对话结束 | 回忆整理者总结，并请求保存许可 |

### 10.4 记忆与边界规则

| 记忆类型 | 例子 | 默认处理 |
|---|---|---|
| 兴趣记忆 | 喜欢戏曲、老电影、种花 | 可保存，但允许修改 |
| 事实记忆 | 曾经做过教师、住过某地 | 保存前简短确认 |
| 情感记忆 | 讲到某段经历很骄傲 | 谨慎保存，避免频繁提起 |
| 敏感记忆 | 故人、疾病、家庭矛盾 | 默认不保存或强确认 |
| 边界偏好 | 不想聊某话题、不喜欢某角色 | 可保存，用于保护用户 |

## 11. 技术实现设想

早期研究阶段可以使用 Wizard-of-Oz 或半自动原型，不必立即完成完整系统开发。如果进入原型阶段，可以采用前后端分离架构：

| 层级 | 技术 | 作用 |
|---|---|---|
| 前端 | React | 多角色聊天界面、记忆卡片、控制按钮 |
| 后端 | FastAPI | 对话接口、实验条件管理、日志保存 |
| 多 agent 编排 | LangGraph 或 AutoGen | 管理角色、状态、发言顺序和边界判断 |
| 数据库 | SQLite 或 PostgreSQL | 保存授权记忆、对话日志和实验 trace |
| 研究后台 | 简单 dashboard | 查看 role trace、topic trace、memory trace、boundary trace |

最小可行原型只需包括：

1. 三种实验条件切换。
2. 2-3 个 AI 角色气泡。
3. 话题卡或图片输入。
4. “换一个人聊”“先别问了”“帮我总结”“不要记住”等低负担控制。
5. 记忆卡片确认。
6. 研究者后台记录每轮使用的角色、话题、记忆和边界判断。

## 12. 预期贡献

### 12.1 理论贡献

提出“关系感知型多智能体陪伴”的视角：老年陪伴中重要的不只是 AI 角色数量，而是关系角色如何随话题、记忆、情绪和边界需求动态变化。

### 12.2 机制贡献

提出 agent-agent conversational cueing：让多个 AI 先短暂互动，再邀请老人加入，把社会性对话本身作为一种记忆线索。

### 12.3 设计贡献

形成面向老年回忆陪伴的设计原则：

- 话题-关系角色匹配
- 低负担角色控制
- 选择性发言和沉默
- 记忆保存确认
- 敏感话题边界守护

### 12.4 方法贡献

提供从访谈到系统编排规则的流程：

> 访谈与共创 → 话题-角色矩阵 → 关系编排规则 → Wizard-of-Oz 原型 → 对比实验 → 内容编码与体验评估

## 13. 可能风险与收缩方案

| 风险 | 表现 | 收缩方式 |
|---|---|---|
| 方向太大 | 同时涉及陪伴、记忆、伦理和技术实现 | 主线聚焦关系编排和社会线索 |
| 多 agent 太复杂 | 老人觉得吵或混乱 | 每轮最多 2-3 个角色，每个只说 1 句 |
| 敏感话题风险 | 故人、疾病、孤独话题可能引发不适 | 前期只作为边界问题，不主动深入 |
| 系统实现压力大 | 完整自动编排耗时 | 先使用 Wizard-of-Oz 或半自动脚本 |
| 评价指标过多 | 分析困难 | 优先聚焦回忆细节、开口压力、陪伴感和控制感 |

## 16. 可向老师汇报的简短版本

本研究计划从“多智能体老年陪伴”进一步收束到三个机制：**关系编排、社会线索引入话题、多智能体角色交互**。我关注的不是简单增加 AI 角色数量，而是研究不同关系角色如何根据话题和用户需求动态出现，例如同龄共鸣者适合时代回忆，晚辈好奇者适合追问细节，中年传承者适合承接经验，边界守护者则在敏感话题中判断是否继续。

方法上，我计划先进行半结构访谈和共创，通过话题卡、角色卡、三种话题引入方式和记忆卡片，了解老年人对不同话题、不同角色和多个 AI 先短暂对话的接受度。访谈结果将转化为话题-关系角色匹配矩阵、社会线索设计规则和记忆边界规则。

后续原型阶段将比较 single-agent、fixed-role multi-agent 和 relationship-aware multi-agent 三种条件，评估它们在回忆细节、自我表达、陪伴感、信任感、控制感和认知负担上的差异。这样，本研究可以在 ReminiBuddy 等已有工作的基础上进一步探索：agent-agent conversation 是否能够成为一种新的社会性记忆线索，以及关系感知的动态角色编排是否更适合老年回忆陪伴场景。
